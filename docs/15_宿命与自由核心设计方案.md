# 宿命与自由核心设计方案

> 日期: 2026-05-30
> 状态: 设计草案
> 用途: 统一账本、DayMap、经营、夜晚服务、NPC 分支与 30 天节奏的主题方向。

---

## 1. 核心体验

《地下城酒馆》的核心不是采集、合成或经营本身，而是：

> 玩家提前看见一个人的命运后，如何用有限的一天、有限的钱和有限的手去回应。

主题句：

> 自由不是消灭宿命，而是在知道代价后仍然选择。

所有系统围绕这个体验服务：

- **账本**预记命运并记录后果。
- **DayMap**让玩家在调查、干预、关系和经营之间取舍。
- **经营**提供资源压力和选择成本。
- **夜晚服务**把白天选择递到客人面前。
- **NPC 结局**记录不同人生，不做善恶审判。

### 1.1 与物理重设计的边界

本文档只定义主题和叙事结构。合成、容器、L1/L2/L3、上菜风格和失败处理以 `docs/13_合成系统物理重设计需求文档.md` 为准。

当前原则：

- “宿命下坠，玩家递出后果”是主题隐喻，不是新物理规则。
- **L1 物品属性**影响宿命类分支。
- **L2 配方/成品**触发记忆、故事和特定 NPC 内容。
- **L3 动作风格**影响信任、反馈和故事是否打开，不默认决定生死。
- Ryan 这类关键命运优先由账本预记、DayMap 选择、剧情变量、L1 属性和是否告知真相决定。

## 2. 设计支柱

### 2.1 账本预记命运

账本是命运载体，不只是结算页。

- 平时记录收入、声望、材料消耗和服务结果。
- 关键日前出现“预记”。
- 玩家改写后，原条目保留划线、旁注或污染痕迹。
- 账本只记录事实，不评价玩家。

示例：

```text
Day 3 预记：
Ryan，北矿道，未归。

旁注：
未赴约。存活。精神失序。
```

### 2.2 白天分配自由

白天是调查与准备阶段。重要剧情日前，至少提供三类互斥选择：

- 查明真相。
- 取得干预工具。
- 接触当事人。

玩家不能全做完。前期建议每天 2 点行动力。

### 2.3 经营制造成本

经营不是最终目标，而是自由的成本。

- 金币用于材料、情报、工具、代采和配方。
- 声望决定谁愿意来、谁愿意信任玩家、谁会带来麻烦。
- 材料制造轻压力，但普通经营失败不能直接让主线崩盘。
- 基础材料易得，剧情材料稀缺且含义明确。

### 2.4 夜晚递出后果

夜晚服务负责把白天选择交给客人。

动作表达可以承载态度：

- 拖拽 = 干预。
- 下坠 = 不可完全控制的趋势。
- 松手 = 放弃控制。
- 接住 = 承担。
- 递给客人 = 交出后果。

限制：避免把手滑误判为道德选择。只有明确上菜区、关键物品和明确递交流程进入叙事判断。

### 2.5 命运档案

结局收集包装为账本中的“命运档案”。

- 重要 NPC: 4 个主要结局 + 1 个隐藏结局。
- 次重要 NPC: 2-3 个结局。
- 普通客人只做群体状态。
- 未解锁结局可用模糊标题，不剧透条件。

### 2.6 30 天压力

不是每天都改命，但每天都应靠近某个命运。

日子类型：

- 普通经营日。
- 预兆日。
- 关键选择日。
- 系统解锁日。
- 收束日。

避免每天都是生死选择，也避免中段只刷材料。

## 3. Ryan 样板线

Ryan 线承担 Day 1-3 的主题教学。

核心问题：

> 为了救一个人，玩家有没有权利剥夺他的选择？

### 3.1 Day 1: 安全教学

目标：

- 教会基础服务流程。
- 建立 Ryan 的亲近感。
- 暗示他将接危险委托。

### 3.2 Day 2: 命运显现

账本预记：

```text
第三日。Ryan。北矿道。未归。
```

白天地点：

| 地点 | 作用 |
|---|---|
| 训练场 | 找 Ryan，可选择是否告知真相 |
| 佣兵告示板 | 调查北矿道委托 |
| 蘑菇洞 | 获取迷睡花粉 |
| 药师摊 | 了解花粉副作用 |
| 商店 | 正常补货，保证夜晚经营 |

夜晚可选方向：

- 正常服务，不干预。
- 使用迷睡花粉，让 Ryan 睡过清晨。
- 告知真相，让 Ryan 自己选择。
- 犹豫、反悔、倒掉特殊饮品。

### 3.3 Day 3: 代价结算

基础结局：

| 路线 | 结果 | 主题 |
|---|---|---|
| 不说、不干预 | Ryan 死亡 | 旁观也是选择 |
| 不说、偷偷下药 | Ryan 存活但崩溃或怨恨 | 改命但剥夺自由 |
| 告诉他、他仍出发 | Ryan 死亡，但清醒赴命 | 尊重自由不等于救命 |
| 告诉他、他不出发 | Ryan 存活，但背负代价 | 自由也有代价 |

隐藏结局只提供更复杂角度，不做“真结局”。

## 4. DayMap 定义

设计目标：

> 白天不是采集，是把有限的一天花给某个人。

地点类型：

| 类型 | 作用 |
|---|---|
| 资源地点 | 获取普通材料或特殊材料 |
| 情报地点 | 获得 NPC 命运线索 |
| 干预地点 | 获得工具、药物、护符、伪造物、传话机会 |
| 关系地点 | 接触 NPC，告知、隐瞒、劝说或建立信任 |
| 商店地点 | 补经营资源，也可出售特殊信息或工具 |

地点信息分三层：

1. 明确信息：行动力消耗、已知收益、地点类型。
2. 暗示信息：与剧情相关但不剧透的描述。
3. 未知信息：访问后揭示的真相、副作用或隐藏关系。

## 5. 经营系统定位

普通经营承担三件事：

1. 让玩家每晚有事可忙。
2. 给白天选择制造机会成本。
3. 在沉重剧情之间提供呼吸感。

### 5.1 普通客人

普通客人负责日常压力：

- 随机订单。
- 耐心倒计时。
- 金币收入。
- 声望变化。
- 世界观闲话。

普通客人不承担重大命运分支，避免主线被日常手滑破坏。

### 5.2 资源

- **金币**是选择空间，不是万能钥匙。
- **声望**代表谁愿意来找玩家，同时带来更多麻烦。
- **配方**是与 NPC 交流的语言，不只是高价商品。

配方示例：

| 配方 | 叙事含义 |
|---|---|
| 麦芽酒 | 冒险者日常 |
| 葡萄酒 | 回忆、悼念 |
| 草药茶 | 安抚、倾听 |
| 肉汤 | 照顾、体力 |
| 香料红酒 | 刺激、逃避 |
| 白眠酒 | 遗忘、强制休息 |

## 6. NPC 模板

每个重要 NPC 回答一个不同主题问题，避免重复 Ryan。

| NPC | 主题问题 |
|---|---|
| Ryan | 救命是否可以剥夺自由 |
| Mira | 逃避责任是否也是自由 |
| 老兵 | 被过去束缚的人如何重新拥有未来 |
| 账本相关 NPC | 相信宿命是否会变成责任逃避 |
| 情报商人 | 当自由变成商品，穷人还自由吗 |
| 学徒/孩子 | 是否有权替尚未成熟的人决定人生 |

设计模板：

```text
名字：
表层身份：
宿命：
想要的自由：
玩家能提供的干预：
干预代价：
是否存在“告知真相”选择：
关键配方：
关键白天地点：
最终问题：
```

## 7. 结局与成就

主 NPC 结局按两个轴设计：

- 玩家是否干预。
- NPC 是否知情/自愿。

|  | NPC 不知情 | NPC 知情 |
|---|---|---|
| 玩家不干预 | 宿命原样发生 | 知情接受宿命 |
| 玩家干预 | 被控制的改命 | 共同选择新路 |

成就标题克制、诗性，不写攻略条件。

示例：

- 未归者
- 替他合上门
- 清醒赴约
- 另一盏熄灭的灯
- 温柔的牢笼

通关可根据玩家长期倾向给出全局称号，例如记账人、见证者、递灯者、改命者。称号只总结方式，不评价善恶。

## 8. 30 天节奏骨架

### 8.1 阶段划分

| 阶段 | 天数 | 作用 |
|---|---|---|
| 第一阶段 | Day 1-3 | Ryan 主题教学 |
| 第二阶段 | Day 4-8 | 日常扩展，Mira 登场，玩家缓冲 |
| 第三阶段 | Day 9-15 | 系统咬合，多条人物线交叉 |
| 第四阶段 | Day 16-24 | 代价累积，前期选择回流 |
| 第五阶段 | Day 25-30 | 收束、最后选择、结局展示 |

### 8.2 日子配比

| 类型 | 预计数量 | 作用 |
|---|---:|---|
| 普通经营日 | 10-12 | 赚钱、配方、呼吸感 |
| 预兆日 | 6-8 | 账本提示，白天调查 |
| 关键选择日 | 6-7 | NPC 命运分支 |
| 解锁/系统日 | 3-4 | 新容器、配方、地图 |
| 收束日 | 2-3 | 结局整理 |

## 9. 落地优先级

1. 先用现有系统重做 Ryan Day 1-3 文案与账本表现。
2. 将 LedgerScreen 扩展为“经营账 + 人情账 + 命运账”。
3. 将 DayMap 扩展为“资源/情报/干预/关系/商店”混合地点。
4. 将普通经营资源改为服务选择空间，而不是纯数值增长。
5. 等重力系统进入 Tavern 后，再接入关键服务动作和 L3 动作风格。

当前阶段不做复杂成就、回跳节点或完整 30 天内容。先做 Ryan 样板线，验证主题是否成立。
