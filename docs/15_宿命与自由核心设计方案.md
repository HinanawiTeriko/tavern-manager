# 宿命与自由核心设计方案

> 日期: 2026-05-29
> 状态: 设计草案
> 目的: 将“宿命与自由”确立为《地下城酒馆》的核心体验轴，指导后续账本、白天、经营、夜晚服务、NPC 结局与 30 天节奏设计。

---

## 1. 核心定位

《地下城酒馆》不是单纯的采集、合成、经营游戏。它的核心体验应当是：

> 当玩家提前看见一个人的命运，会用自己有限的一天、有限的钱、有限的手，怎样对待他。

游戏的主题不是“玩家战胜宿命”，而是：

> 自由不是消灭宿命，而是在知道代价后仍然选择。

重力系统、账本、DayMap、经营资源和 NPC 分支都应围绕这个主题服务。

---

## 2. 六个设计支柱

### 2.1 账本预记命运

账本不是普通结算页，而是命运的载体。

- 平时记录收入、声望、材料消耗和服务结果。
- 关键日提前出现“预记”：某个 NPC 将在未来遭遇某种命运。
- 玩家选择后，账本不删除原命运，而是在原条目上留下划线、旁注、污染、改写痕迹。
- 账本只记录事实，不做道德审判。

示例：

```text
Day 3 预记：
Ryan，北矿道，未归。
```

如果玩家偷偷让 Ryan 睡过清晨：

```text
Ryan，北矿道，未归。  [划去]
旁注：未赴约。存活。精神失序。
```

### 2.2 白天分配自由

白天不应只是采集阶段，而是“调查与准备阶段”。

玩家用有限行动力决定今天把时间花在哪里：

- 查明真相
- 获取干预工具
- 接触当事人
- 补充经营资源
- 购买情报或道具

设计原则：

> 每个重要剧情日前，白天至少提供三种选择：查明真相、取得干预工具、接触当事人。玩家不能全做完。

### 2.3 经营制造成本

经营不是最终目标，而是自由的成本。

- 金币让玩家购买材料、情报、工具、代采服务和特殊道具。
- 声望决定谁愿意来酒馆、谁愿意相信玩家、谁会把麻烦带到玩家面前。
- 材料消耗制造轻压力，但不能让玩家因普通经营失败而主线崩盘。
- 基础材料易得，剧情材料稀缺且意义明确。

玩家应经常面对这种拉扯：

> 我想去调查 Ryan 的命运，但今晚也缺材料。
> 我想买药，但还要攒钱解锁配方。
> 我想告诉真相，但一天只有两个行动。

### 2.4 夜晚递出后果

夜晚服务是玩家把白天选择交到客人面前的过程。

未来重力系统接入后，核心隐喻是：

> 宿命下坠，玩家递出后果。

重力不是单纯判分器，而是主题表达：

- 拖拽代表干预。
- 下坠代表不可完全控制的趋势。
- 松手代表放弃控制。
- 接住代表承担。
- 递给客人代表把后果交出去。

系统必须避免把手滑误判为道德选择。只有在明确上菜区、关键物品、明确递交流程中，动作才进入叙事判断。

### 2.5 命运档案记录人生

结局收集可以存在，但应包装为账本中的“命运档案”，而不是普通成就墙。

- 重要 NPC 有 4 个主要结局 + 1 个隐藏结局。
- 次重要 NPC 有 2-3 个结局。
- 普通客人不做个人结局，只做群体状态。
- 未解锁结局用模糊标题或残缺文字提示，不直接剧透条件。
- 结局不是奖杯，而是“另一种人生的记录”。

设计目标：

> 让玩家意识到每个人都不只有一种命运，但每一种命运都有人付账。

### 2.6 30 天节奏推进压力

不是每天都改命，但每天都在靠近某个命运。

30 天应交替安排：

- 普通经营日
- 预兆日
- 关键选择日
- 系统解锁日
- 收束日

避免两个极端：

- 每天都有重大生死选择，导致玩家麻木。
- 只有少数几天有剧情，导致中段像刷材料。

理想体验：

> 有些日子只是开店。
> 但玩家知道，账本某一页的墨迹正在慢慢变深。

## 3. Ryan 样板线

Ryan 线是主题教学样板，承担前 3 天体验。

### 3.1 主题问题

> 为了救一个人，玩家有没有权利剥夺他的选择？

Ryan 的核心不是“救不救”，而是：

> 救命不等于给自由。

### 3.2 Day 1: 安全教学日

目标：

- 教会基础服务流程。
- 建立 Ryan 的亲近感。
- 让玩家觉得他有希望、有点冒失、但值得在意。

建议内容：

- 白天只做简单准备。
- 夜晚普通客人教学。
- Ryan 登场，点简单饮品。
- 服务后提到明天或后天要去危险委托，但语气轻松。

### 3.3 Day 2: 命运显现日

账本第一次出现预记：

```text
第三日。Ryan。北矿道。未归。
```

白天 DayMap 出现有限地点：

| 地点 | 作用 |
|---|---|
| 训练场 | 找 Ryan，可选择是否告知真相 |
| 佣兵告示板 | 调查北矿道委托 |
| 蘑菇洞 | 获取迷睡花粉 |
| 药师摊 | 了解花粉副作用 |
| 商店 | 正常补货，保证夜晚经营 |

玩家只有有限行动力，无法全部完成。

夜晚 Ryan 到店。玩家可以：

- 正常服务，不干预。
- 使用迷睡花粉，偷偷让他睡过清晨。
- 告知真相后，让 Ryan 自己选择。
- 犹豫、反悔、倒掉特殊饮品。

### 3.4 Day 3: 代价结算日

Ryan 线至少提供四个基础结局：

| 路线 | 结果 | 主题 |
|---|---|---|
| 不说、不干预 | Ryan 死亡 | 旁观也是选择 |
| 不说、偷偷下药 | Ryan 存活但崩溃或怨恨 | 改变宿命，但剥夺自由 |
| 告诉他、他仍出发 | Ryan 死亡，但清醒赴命 | 尊重自由，不等于拯救生命 |
| 告诉他、他不出发 | Ryan 存活，但背负羞耻或代价 | 自由也不是无代价的 |

可预留隐藏结局：

- Ryan 活下来了，但另一个人替他死。
- Ryan 死了，但留下的信在后期救了别人。
- Ryan 活下来，并在后期关键日救了某个 NPC。

隐藏结局不应是“真结局”，而是更复杂的命运角度。

## 4. DayMap 重新定义

DayMap 的设计目标：

> 白天不是采集，是把有限的一天花给某个人。

### 4.1 地点类型

| 类型 | 作用 |
|---|---|
| 资源地点 | 获取普通材料或特殊材料 |
| 情报地点 | 获得 NPC 命运线索 |
| 干预地点 | 获得工具、药物、护符、伪造物、传话机会 |
| 关系地点 | 直接接触 NPC，告知、隐瞒、劝说或建立信任 |
| 商店地点 | 补经营资源，也可出售特殊信息或工具 |

### 4.2 信息显示

地点信息分三层：

1. 明确信息：行动力消耗、已知收益、地点类型。
2. 暗示信息：一句与剧情相关但不剧透的描述。
3. 未知信息：访问后才揭示的真相、副作用、隐藏关系。

示例：

```text
蘑菇洞
消耗 1 行动力
已知：草药、白眠花
线索：药师说它能让人睡过最糟的一夜。
```

### 4.3 行动力

前期建议每天 2 点行动力。

2 点行动力能制造清晰取舍：

- 调查 + 找人
- 找人 + 拿工具
- 拿工具 + 补经营
- 两个都资源

后期可通过声望、雇人代采、酒馆升级等方式扩展选择空间，但不宜过早给太多行动力。

## 5. 经营系统定位

普通经营承担三件事：

1. 让玩家每晚有事可忙。
2. 给白天选择制造机会成本。
3. 在沉重剧情之间提供呼吸感。

### 5.1 普通客人

普通客人负责日常压力：

- 随机订单
- 耐心倒计时
- 金币收入
- 声望变化
- 世界观闲话

普通客人不承担重大剧情分支，避免主线被日常手滑破坏。

### 5.2 金币

金币不是单纯强度资源，而是选择空间：

- 买材料
- 买情报
- 买特殊工具
- 雇人代采
- 解锁配方

但金币不能成为万能钥匙。某些选择必须通过时间、关系或亲自见面获得。

### 5.3 声望

声望不是分数，而是“谁愿意来找你”。

- 声望越高，越多重要 NPC 和特殊客人出现。
- 声望越高，玩家也越可能被卷入更多麻烦。

高声望应同时是奖励和负担。

### 5.4 配方

配方是与 NPC 交流的语言，不只是高价商品。

示例定位：

| 配方 | 情绪/叙事含义 |
|---|---|
| 麦芽酒 | 直接、粗糙、冒险者日常 |
| 葡萄酒 | 回忆、悼念、旧事 |
| 草药茶 | 安抚、倾听、恢复 |
| 肉汤 | 照顾、体力、家常感 |
| 香料红酒 | 刺激、逃避、夜晚情绪 |
| 白眠酒 | 遗忘、错过、强制休息 |

## 6. NPC 群像模板

每个重要 NPC 都应回答一个不同的主题问题，避免重复 Ryan。

推荐阵容：

| NPC | 主题问题 |
|---|---|
| Ryan | 救命是否可以剥夺自由 |
| Mira | 逃避责任是否也是自由 |
| 老兵 | 被过去束缚的人如何重新拥有未来 |
| 账本相关 NPC | 相信宿命是否会变成责任逃避 |
| 情报商人 | 当自由变成商品，穷人还自由吗 |
| 学徒/孩子 | 是否有权替尚未成熟的人决定人生 |

NPC 设计模板：

```text
名字：
表层身份：
他/她的宿命：
他/她想要的自由：
玩家能提供的干预：
干预的代价：
是否存在“告知真相”选择：
关键配方：
关键白天地点：
最终问题：
```

## 7. 命运档案与成就

结局收集应由账本承载。

### 7.1 结局矩阵

主 NPC 的基础结局可按两个轴设计：

- 玩家是否干预。
- NPC 是否知情/自愿。

|  | NPC 不知情 | NPC 知情 |
|---|---|---|
| 玩家不干预 | 宿命原样发生 | 知情接受宿命 |
| 玩家干预 | 被控制的改命 | 共同选择新路 |

### 7.2 成就文案

成就标题应克制、诗性，不写攻略条件。

示例：

- 未归者
- 替他合上门
- 清醒赴约
- 另一盏熄灭的灯
- 温柔的牢笼

### 7.3 全局命运风格

通关时可根据玩家长期选择标签给出全局称号。

| 倾向 | 称号示例 |
|---|---|
| 旁观最多 | 记账人 |
| 控制最多 | 温柔的牢笼 |
| 告知最多 | 见证者 |
| 共担最多 | 递灯者 |
| 转移最多 | 改命者 |

这些称号不应评价善恶，只总结玩家面对命运的方式。

## 8. 30 天节奏骨架

### 8.1 阶段划分

| 阶段 | 天数 | 作用 |
|---|---|---|
| 第一阶段 | Day 1-3 | Ryan 主题教学 |
| 第二阶段 | Day 4-8 | 日常扩展，Mira 登场，玩家缓冲 |
| 第三阶段 | Day 9-15 | 系统咬合，多条人物线交叉 |
| 第四阶段 | Day 16-24 | 代价累积，前期选择回流 |
| 第五阶段 | Day 25-30 | 收束、最后选择、结局展示 |

### 8.2 日子类型

| 类型 | 预计数量 | 作用 |
|---|---:|---|
| 普通经营日 | 10-12 | 赚钱、配方、呼吸感 |
| 预兆日 | 6-8 | 账本提示，白天调查 |
| 关键选择日 | 6-7 | NPC 命运分支 |
| 解锁/系统日 | 3-4 | 新容器、配方、地图 |
| 收束日 | 2-3 | 结局整理 |

| 天数 | 类型 | 内容 |

## 9. 后续落地建议

优先级建议：

1. 先用现有系统重做 Ryan Day 1-3 的文案与账本表现，不依赖重力主界面。
2. 将 LedgerScreen 扩展为“经营账 + 人情账 + 命运账”三层结构。
3. 将 DayMap 从采集列表扩展为“资源/情报/干预/关系/商店”混合地点。
4. 将普通经营资源改为服务选择空间，而不是纯数值增长。
5. 等重力系统正式进入 Tavern 后，再接入关键服务动作和 L3 动作风格。

当前阶段不建议急着实现复杂成就、回跳节点或完整 30 天内容。先做 Ryan 样板线，验证主题是否成立。
