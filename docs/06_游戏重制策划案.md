# 地下城酒馆 2D 重设计·策划案

> 日期: 2026-05-24 | 状态: 已实施 | 工期: 约 1 个月

---

## 1. 游戏总览

**游戏名称**：地下城酒馆

**核心概念**：你在吧台后面经营一间地牢酒馆，招待冒险者。白天派人采集材料，晚上在吧台后为客人调酒上菜。30 天里，几位重要 NPC 的命运在你手中交织——而你的故事也在其中。

**主题**：宿命与自由。每个重要 NPC 都背负着某种"注定"的命运，玩家的选择可能打破它，也可能见证它。

**画面风格**：2D，VA-11 Hall-A 式吧台视角（面朝客人），1280×720。

**技术**：Godot 4.6 .NET，Dialogue Manager 插件驱动 NPC 对话。

**参考作品**：VA-11 Hall-A（吧台氛围）、潜水员戴夫（经营循环）、赛博朋克酒保行动（对话节奏）。

---

## 2. 场景结构

4 个场景：

| 场景 | 说明 |
|------|------|
| 标题界面 | 沿用现有，微调美术 |
| 白天采集 | 菜单/地图式，选派遣地点、消耗体力、看结果报告 |
| 晚上酒馆 | 核心场景——吧台视角，合成、服务、客人对话 |
| 结局结算 | 30 天后展示各 NPC 命运、经营总成绩 |

- 白天采集是纯 UI 界面，不涉及角色操控。
- 晚上酒馆是主场景，固定吧台画面。
- 所有场景 2D，1280×720。

---

## 3. 核心循环

每天两阶段：

```
白天（采集阶段）
  → N 点体力，自行分配去不同采集点
  → 每处消耗体力，获得材料/剧情道具
  → 可能触发 NPC 偶遇事件（简短对话）
  → [时间推进]

夜晚（经营阶段）
  → 吧台视角，客人陆续到来
  → 普通客人随机点单 → 合成 → 上菜 → 收钱
  → 重要 NPC 在特定天数来访 → 对话 + 点单
  → 最后一个客人离开 → 营业结束

结算 → 进入下一天
```

- 白天约 2-3 个决策点，快速过渡。
- 夜晚是核心，普通客人提供经营压力，重要 NPC 提供叙事推进。

## 4. NPC 叙事系统

### 规模

3-4 个重要 NPC，每人 4-5 个关键场景，分散在 30 天中。

### 网状关系

NPC 之间不孤立。NPC A 的故事发展影响 NPC B 的后续对话和可用分支。改变 A 的命运，B 的结局也可能随之变化。

### 叙事结构

每个 NPC：起点（带着命运出现）→ 转折（玩家介入点）→ 终点（30 天结局）。

- 玩家选择影响结局方向，每名 NPC 有 1-2 个关键时刻。
- 所有 .dialogue 文件由 Dialogue Manager 插件驱动，沿用现有技术栈。

### 变量与分支

沿用现有 d2/d3 的变量机制（如 `has_sleep_powder`），扩展为全局叙事变量池：
- 关键道具持有（如迷睡花粉）
- NPC 好感/信任值
- 关键抉择标记（是否干预某事件）
- 跨 NPC 连锁变量（A 的结局标记影响 B 的可用分支）

### 呈现

- 重要 NPC 坐在吧台对面，有立绘/表情变化。
- 普通客人只有文字需求框，不占叙事带宽。

## 5. 合成系统

### 四种手势

| 手势 | 操作 | 用于 |
|------|------|------|
| 拖拽组合 | 材料 A 拖到材料 B | 基础合成 |
| 摇晃 | 按住容器左右/上下晃动 | 饮品摇匀 |
| 加热 | 材料拖到火源放置片刻 | 煮/烤 |
| 搅拌 | 在容器内画圈搅动 | 汤、混合品类 |

### 流程

客人点单 → 选材料 → 按配方执行手势 → 成品出现（动画+音效）→ 拖到客人面前上菜。

### 视觉呈现

- 合成过程有粒子/微动画（倒酒有液体、加热有蒸汽）。
- 成品出现在吧台上，有亮相感。
- 上菜时客人有表情/动作反馈。

### 配方

沿用现有 9 种基础配方，视工期扩展 2-3 种新配方。

## 6. 采集系统

纯菜单/地图式，不涉及角色操控。

### 流程

- 展示地牢区域静态地图 + 地点标记。
- 每个地点标注可获得的材料类型和消耗体力值。
- 玩家有固定体力上限（5 点），自行分配。
- 点击"出发"展示结果：每个地点获得什么、数量多少。
- 可能触发 NPC 偶遇事件。

### 采集点（初定 4-5 个）

| 地点 | 主要产出 |
|------|----------|
| 菌菇林地 | 草药、蘑菇 |
| 废弃矿道 | 麦芽（野麦）、矿石 |
| 暗河沿岸 | 生肉（捕鱼/小兽） |
| 葡萄藤架 | 葡萄 |
| 农庄磨坊 | 面粉 |

### 产出类型

| 类型 | 说明 | 例子 |
|------|------|------|
| 常规材料 | 合成配方用，每处固定产出 | 麦芽、葡萄、草药、生肉、面粉 |
| 剧情道具 | 特定天数/条件解锁，出现在指定采集点 | 迷睡花粉（d2 触发后出现） |

- 剧情道具平时不可见，关键节点才解锁。
- 有些道具通过 NPC 对话获取，无需采集。

### 扩展空间

后续可增加新地点、稀有材料概率、工具升级等。

---

## 7. 客人系统

### 普通客人

- 随机生成，随机名池（12 个）。
- 短暂停留 → 点单需求框 → 合成 → 上菜 → 付钱离开。
- 耐心倒计时（60s），超时离开。
- 纯经营压力 + 收入来源。

### 重要 NPC

- 特定天数固定来访，有立绘和表情。
- 先对话（.dialogue 驱动），对话中可能点单。
- 玩家做的饮品/食物影响对话走向（做对加分，做错有反应）。
- 对话结束后离开，可能影响后续其他 NPC 剧情。

## 8. 经济与进度

### 双货币

| | 金币 | 声望 |
|------|------|------|
| 获取 | 客人付钱、NPC 关键事件 | 服务正确、NPC 好感、特殊事件 |
| 用途 | 购买材料、解锁采集点 | 解锁配方、触发高级 NPC 剧情 |

### 酒馆等级（3 级）

声望达标自动升级。每级解锁新配方、新采集点、新 NPC 来访条件。不需要建筑/装修系统。

### 30 天进度曲线

| 阶段 | 天数 | 内容 |
|------|------|------|
| 教学引入 | 1-3 | 首位 NPC 登场，基础合成教学 |
| 展开期 | 4-10 | 核心 NPC 陆续登场，配方逐步解锁 |
| 冲突期 | 11-20 | 关键剧情抉择密集出现，网状交叉显现 |
| 收束期 | 21-28 | 故事收束，高级配方可用，经营压力增大 |
| 结局 | 29-30 | 最终结算，各 NPC 结局定格 |

## 9. 技术架构

基于现有代码演进，不推翻重来。

### 文件结构（符合现有规范）

```
scripts/
  autoload/
    GameManager.cs       # 核心协调，从 242 行拆出子系统逻辑
  systems/
    NarrativeManager.cs  # NPC 故事状态、变量读写、天数推进
    CraftSystem.cs       # 合成逻辑 + 手势检测
    GuestSystem.cs       # 客人管理（普通+NPC）
    EconomySystem.cs     # 金币/声望/等级
    DayCycleSystem.cs    # 昼夜切换、体力分配
  ui/
    TavernView.cs        # 晚上酒馆主界面（吧台视角）
    DayMapView.cs        # 白天采集地图界面
    CraftStation.cs      # 合成操作区 UI（手势区域）
    EndingScreen.cs      # 结局结算界面
    TitleScreen.cs       # 沿用现有
  components/            # 挂载节点脚本（按需）
```

### 关键改造

- `GameManager.cs` 拆出子系统，保留 Autoload 协调角色。
- 配方数据从硬编码字典迁移到 JSON 资源文件。
- Dialogue Manager 插件沿用，.dialogue 文件继续用于 NPC 对话。
- 现有 `d1~d3.dialogue` 保留，整合进新 NarrativeManager。

### 数据流

```
GameManager (协调)
  ├─ DayCycleSystem → (白天/夜晚切换)
  ├─ NarrativeManager → (对话状态、变量)
  ├─ CraftSystem → (手势识别、配方匹配)
  ├─ GuestSystem → (客人队列、耐心计时)
  └─ EconomySystem → (金币、声望、等级)
```
