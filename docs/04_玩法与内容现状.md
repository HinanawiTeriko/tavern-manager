# 玩法与内容现状

## 游戏定位

玩家经营一间地下城酒馆。白天在地图上选择行动，夜晚根据传闻和库存制定菜单，用物理吧台制作订单，并在结算中看到经营、口碑和人物命运的后果。

核心体验不是“点按钮合成”，而是：

- 白天用有限行动收集材料、情报和证据。
- 夜晚根据传闻选择菜单。
- 在吧台上拖拽、投料、摇晃、按压、搅拌、调味。
- 服务不同客人，积累金币、声望、常客记忆和故事变化。

## 当前玩家流程

```text
新游戏/继续
  -> DayMap
     - 采集材料
     - 访问商店
     - 听传闻
     - 推进调查/剧情
  -> Tavern
     - 查看传闻和昨日回响
     - 制定今日菜单
     - 服务重要 NPC、常客、客群客人
  -> LedgerScreen
     - 看结算、客人队列、传闻摘要、命运提示
     - 可重写今日
```

## DayMap

DayMap 地点定义在 `data/locations.json`。当前地点类型包含：

- 采集地点：奖励材料。
- 商店：购买材料、配方、能力。
- 贴文/委托地点：推进 Ryan/Toby 等线索。
- 调查入口：进入独立调查场景。
- 关系/剧情地点：Mira stall、fixer 等。
- 传闻来源：访问后写入当天传闻池。

获得传闻时，DayMap 会使用顶部 `GatheringToast` 提示“听到传闻”，完整内容仍进入返回消息和菜单准备数据。

## 菜单准备

夜晚开始前先配置菜单。菜单准备面板读取：

- 今日传闻：推荐标签、可能客群、受影响常客。
- 昨日回响：常客记忆、口碑偏向。
- 当前可上菜产品：来自 `CraftSystem.get_orderable_products(day)`。

顾客只会从当日菜单和系统允许订单里选择。菜单越窄，经营越可控，但也可能错过传闻客群。

## 客人

当前有三类客人：

| 类型 | 来源 | 说明 |
|---|---|---|
| 重要 NPC | `NarrativeManager`、`RyanSliceSystem` | 推进主线、好感、结局 |
| 常客 | `data/regular_customers.json` | 30 名，带三表情立绘、偏好、记忆、特质 |
| 客群临时客 | `data/guest_group_profiles.json` | 受传闻影响，使用客群偏好和复用立绘池 |

常客有稳定 `npc_id`，客群临时客没有稳定 `npc_id`，但有 `guest_group` 和 `portrait_id` metadata。

## 传闻和胃口

传闻系统的作用：

- 从 DayMap 地点获得。
- 当天菜单准备中显示推荐。
- 影响当晚客群/常客出现权重。
- 常客胃口命中时形成“传闻应验”记忆。
- 结算屏汇总命中、错过和受影响客人。

胃口由 `AppetiteSystem` 根据菜品标签和顾客偏好评分。菜品标签来自 `data/food_attributes.json`。

## 物理烹饪

Tavern 工作面当前包含：

- 快捷栏取材。
- `DeskItem` 物理拖拽。
- 酒桶：投料、摇晃、产出。
- 烤架：按压加热、熟/焦反馈。
- 炖锅：投料、勺子搅拌。
- 调味：香料影响菜品属性。
- 上菜：成品拖入顾客投放区。

上菜动作会生成 craft style，供重要 NPC 反馈使用。

## 剧情现状

当前已接入的主要人物/路线：

- Ryan：Day 1-3 竖切、伪装身份、证据/替代委托、命运结果。
- Mira/Toby：Mira stall、Toby contract、责任推理、关系阈值。
- Evelyn/grey ledger：账本缺口、公账推理、Day 20 收束。

剧情系统仍在扩展中。普通客人系统目前主要影响经营、菜单、口碑和信息压力，不直接替代重要 NPC 分支。

## 结算和重写今日

LedgerScreen 展示每日服务结果，并提供“重写今日”。重写今日恢复日初快照，包括当天获得的传闻，所以重开后需要重新访问地点才能再次获得该传闻。
