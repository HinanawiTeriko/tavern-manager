## 一、项目定位

**地下城酒馆** — Godot 4.6 .NET **3D** 模拟经营游戏。

玩家扮演一间地牢深处小酒馆的老板，在白天采集食材、与商人交易、和冒险者对话接取任务；在夜晚营业服务客人，合成食物饮料，赚取金币与声望，逐步将破败酒馆经营成地下城最著名的据点。

- **引擎**: Godot 4.6.x (.NET/Mono)
- **渲染**: Forward+ (3D)
- **语言**: C# (.NET 8.0)
- **视角**: 第三人称自由移动 + 固定视角交互
- **分辨率**: 1280×720（可缩放）
- **项目结构**: `scripts/` (C#)、`scenes/` (.tscn)、`assets/` (3D模型/贴图/音频/字体)、`data/` (配表)、`docs/` (文档)

## 二、核心玩法循环

```
        白天（准备阶段）                    夜晚（营业阶段）
   ┌──────────────────────┐      ┌──────────────────────┐
   │ · 在酒馆内自由移动      │      │ · 客人陆续到来          │
   │ · 与 NPC/商人对话       │ ───→ │ · 根据订单合成食物/饮料   │
   │ · 采集/购买食材          │      │ · 服务客人赚取金币/声望   │
   │ · 接取任务              │      │ · 客人离开（满意/不满）   │
   │ · 装修/扩建酒馆          │      │ · 特殊事件触发          │
   └──────────────────────┘      └──────────────────────┘
          ↑                              │
          └──────── 隔天循环 ←────────────┘
```

## 四、各系统详细设计

### 4.1 背包系统 (InventorySystem)

```
物品数据结构:
{
  "id": "herb_01",           // 唯一物品ID
  "key": "Herb",             // 物品类型Key
  "name": "暗影草",           // 显示名
  "type": "Material",        // Material / Food / Tool / Quest / Furniture
  "rarity": 1,               // 1普通 2稀有 3史诗
  "stack": 12,               // 当前堆叠数
  "maxStack": 99,            // 最大堆叠
  "icon": "res://assets/textures/items/herb_01.png",
  "desc": "生长在地牢深处的发光草药",
  "tags": ["herb", "crafting"]
}
```

- **容量**: 初始 20 格，可通过背包扩容道具提升至 60 格
- **快捷栏**: 10 格（底部 UI），对应数字键 1-0
- **操作**: 拾取/丢弃/使用/拆分/排序/搜索

### 4.2 经济系统 (EconomySystem)

- **货币**: 金币（主要）、声望（经验值，提升酒馆等级）
- **物价**: 基于酒馆等级的浮动物价（1级±5%，5级±25%）
- **收入来源**: 服务客人、完成任务、出售物品
- **支出**: 购买食材、装修扩建、支付工资（后期可雇佣 NPC 帮工）
- **酒馆等级**: 声望累积升级，解锁新配方/区域/商人

### 4.3 采集系统 (GatheringSystem)

- **采集点**: 酒馆周边的地牢区域（蘑菇洞、地下暗河、废弃矿道等小场景）
- **刷新机制**: 每天刷新一次，稀有材料有概率出现
- **工具**: 采集匕首（草药）、矿镐（矿石）、钓竿（鱼类）
- **体力限制**: 每天有采集次数上限（可通过升级提升）

### 4.4 交易系统 (TradeSystem)

- **商人类型**: 旅行商人（随机出现）、固定供应商（每天营业）、黑市商人（特殊物品）
- **交易方式**: 金币购买、以物易物、讨价还价（基于好感度）
- **限时商品**: 每周刷新特价商品
- **出售**: 玩家可将多余物品卖给商人（回收价为基础价的 30%-60%）

### 4.5 合成系统 (CraftSystem)

- **配方**: 保留现有 9 种基础配方，扩展到 20+ 种（含 3D 特色：炖锅、烤炉、调酒台不同设备）
- **合成设备**: 吧台（饮品）、灶台（食物）、烤炉（烤肉/面包）
- **熟练度**: 每种配方有熟练度，满级后产出品质提升
- **失败率**: 新材料组合有失败概率，熟练度越高越低
- **合成队列**: 可排队多个合成任务（后期升级设备解锁）

### 4.6 对话系统 (DialogueSystem)

- **引擎**: 沿用 Dialogue Manager 插件（已安装）
- **现有对话**: `dialogue/d1.dialogue` ~ `d3.dialogue`（莱恩的 3 天叙事线，保留）
- **C# 集成**: DialogueSystem 封装 Dialogue Manager 调用，通过信号通知游戏逻辑
- **触发方式**: 靠近 NPC 按 F 键、自动触发（剧情）、对话选项
- **分支**: 对话选项可影响好感度、任务走向、剧情分支
- **变量系统**: 支持条件判断（如 `has_sleep_powder` → 影响 d3 的结局）

### 4.7 移动系统 (PlayerController / NPCController)

**主角**:
- WASD 移动，鼠标旋转视角（第三人称）
- Space 跳跃（酒馆内可能有高低差）
- E 键交互（与物体/NPC/门）
- F 键对话
- Tab 打开/关闭背包
- 鼠标左键拖拽（合成时）

**NPC**:
- 基于 NavigationAgent3D 的寻路
- 日常作息表（白天在特定区域活动，晚上在酒馆就座）
- 与玩家碰撞检测、自动避让
- 状态: Idle / Patrol / Walking / Sitting / Talking

### 4.8 地图系统 (TavernMapSystem)

- **主地图**: 酒馆内部（3D 场景）
- **区域划分**:
  - 吧台区：合成饮品、服务客人
  - 用餐区：客人就座、上菜
  - 厨房区：烹饪食物
  - 储物区：存放食材、工具
  - 门口区：客人进出、商人停留
  - 地下室（后期解锁）：酒窖、特殊合成
- **子场景**: 采集区域（蘑菇洞、暗河、矿道——通过酒馆后门进入）
- **可交互物**: 门、宝箱、合成设备、储物箱、告示板（任务）、商人摊位

### 4.9 时间系统 (TimeSystem)

- **一天 = 营业前(白日) + 营业中(夜晚) 两个阶段**
- **白日** (6:00-18:00): 自由移动、采集、购物、对话、装修
- **夜晚** (18:00-24:00): 营业，客人到访，合成与服务
- **深夜** (24:00-6:00): 自动过渡，结算收入，推进日期
- **日期推进**: 每天结束后结算、刷新采集点/商人商品/事件

### 4.10 任务系统 (QuestSystem)

- **任务类型**: 主线（酒馆成长故事）、支线（NPC 个人故事）、每日（重复任务）
- **任务结构**: 接取 → 条件追踪 → 完成 → 领取奖励
- **奖励**: 金币、声望、稀有物品、配方解锁
- **任务链**: 以现有莱恩叙事线（d1-d3）为核心主线起点

### 4.11 事件系统 (EventSystem)

- **随机事件**: 酒馆斗殴、名人到访、食材大减价、地牢怪物入侵
- **节日**: 满月节（客人加倍）、丰收日（材料半价）
- **触发条件**: 日期/酒馆等级/已完成任务/好感度组合

### 4.12 装修系统 (BuildSystem)

- **扩建**: 解锁新区域（厨房、地下室、二楼客房）
- **家具**: 购买/摆放桌椅、装饰品、设备
- **效果**: 家具影响客人舒适度（更多小费）、合成效率、容量

### 4.13 账本系统 (LedgerSystem)

账本是本项目的**核心叙事载体**。玩家拥有一本破旧的皮质账本，摆放在吧台一角。它既是经营工具，也是主角的个人日志，承载着酒馆的过去与未来。

**基础功能（经营层面）**:

- **收支明细**: 自动记录每日的金币收支（营业额、进货支出、装修花费、任务报酬等）
- **分类统计**: 按类别筛选（食物收入/饮品收入/材料支出/工资支出/其他）
- **盈亏走势**: 7日/30日收支折线图
- **税务记录**: 每月需向地牢领主缴税，账本记录税务状况（同时也是剧情触发点）

**叙事功能（剧情层面）**:

- **主角日记**: 每天结束后，账本会自动生成一段简短的日志文字（基于当天发生的事件、遇到的人、完成的任务），以主角第一人称口吻书写
- **神秘记录**: 账本前几页被撕掉了，随着剧情推进，偶尔会出现褪色字迹——前任酒馆老板留下的秘密记录，揭示酒馆隐藏的历史
- **客人留言**: 特殊客人离开后，可能在账本中留下签名、涂鸦或暗号
- **债条记录**: 部分 NPC 会向玩家借钱/赊账，逾期不还会触发追债任务
- **线索收集**: 主线剧情的线索碎片以账本条目形式累积，玩家可随时翻阅回顾

**账本交互**:

- 点击吧台上的账本模型 → 进入翻阅模式（书本翻页动画）
- 左侧页码目录、右侧内容区
- 支持标记/书签（玩家可手动标记重要条目）
- 可书写：少量自由文本输入（给未来自己的备注）

**账本在剧情中的角色（示例）**:

```
第 1 天 → 账本自动记录第一笔收入
第 3 天 → 翻阅时发现前任老板留下的褪色字："地窖最深处，别打开那扇铁门"
第 7 天 → 税务官到访，账本成为报税凭证
第 10 天 → 莱恩在账本上签字，成为酒馆的常客记录
第 15 天 → 账本中被撕掉的页面突然复原了一页——揭示一段尘封的往事
```

## 七、客人系统（拓展）

### 7.1 客人类别

| 类别 | 示例 | 特点 |
|------|------|------|
| 普通冒险者 | 战士、法师、盗贼 | 随机订单，标准耐心 |
| 高级冒险者 | 圣骑士、大法师 | 高价订单，高要求 |
| 特殊客人 | 贵族、吟游诗人 | 触发事件/任务 |
| 熟人 | 莱恩等剧情NPC | 对话丰富，有个人故事线 |

### 7.2 客人生成规则

- **生成频率**: 夜晚营业期间每 2-5 秒到访一位
- **客人上限**: 取决于可用座位数
- **耐心时间**: 60 秒（基础），受酒馆等级/舒适度加成

## 十一、AI 操作注意事项

1. **不要修改 `addons/` 下的任何文件**（dialogue_manager 和 godot_mcp 是外部插件）
2. **新增功能必须按系统拆分脚本，不要全部塞进 GameManager**
3. **所有 `res://` 路径必须完整**
4. **遵循 `01_AI开发规范.md` 的目录规范**：脚本放 `scripts/`、场景放 `scenes/`、资源放 `assets/`
5. **3D 场景使用 Forward+ 渲染器，节点基类使用 Node3D**
6. **系统间通过信号（Signal）或 Autoload 方法调用解耦，不要直接引用**
7. **纯代码 3D 模型可以用 CSG 节点（CSGBox3D 等）做白模原型，美术替换后再换 MeshInstance3D**
8. **备份 `project.godot` 后再修改 Autoload/渲染器配置**
