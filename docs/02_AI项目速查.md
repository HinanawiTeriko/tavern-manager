# 地下城酒馆 — AI 项目速查

> 供 AI 快速理解当前项目。本文按代码现状更新，旧的 CraftStation/固定合成槽/SeasoningZone 流程已不再是 Tavern 主流程。

---

## 一、项目定位

**地下城酒馆** 是 Godot 4.6 standard、GDScript、2D GL Compatibility 的吧台模拟经营原型。

玩家白天准备，夜晚在物理吧台上用三种容器制作订单。物理动作不是纯表现：上菜速度会进入 L3 动作风格系统，影响 NPC 反馈和信任。

| 项 | 当前值 |
|----|--------|
| 主场景 | `res://scenes/ui/TitleScreen.tscn` |
| 引擎 | Godot 4.6.x standard，非 .NET/Mono |
| 渲染 | GL Compatibility 2D |
| 分辨率 | 1280×720 |
| Autoload | `GameManager`、`DialogueManager`、`TutorialManager` |
| 插件 | Dialogue Manager、Godot MCP |

## 二、当前玩法循环

```text
TitleScreen
  -> DayMap：选目的地、采集、触发 NPC、整理携带材料
  -> Tavern：从 ShortcutBar 取材，物理加工，拖拽上菜
  -> LedgerScreen：查看当日结算，进入下一天
  -> EndingScreen：第 30 天后结局
```

Tavern 的主循环：

1. 客人出现并提出订单。
2. 玩家从 `ShortcutBar` 点击材料，生成 `DeskItem` 物理体。
3. 材料进入酒桶、烤架或炖锅。
4. 容器按 `container + ingredients` 查询配方并生成成品。
5. 成品拖入 `CustomerDropArea` 松手，上菜结算。
6. `GameManager.request_serve()` 统一路由经济、客人、叙事和 L3 风格。

## 七、容器交互

### 取材与拖拽

- `BarWorkspace` 在 `_init_material_slots()` 中读取库存，把非 product 物品填入快捷栏。
- 左键点击快捷栏槽位会在鼠标处生成 `DeskItem`。
- `DragController` 用 PinJoint 钉住物体；松手后物体继续受重力和惯性影响。
- 成品拖到 `CustomerDropArea` 松手后调用 `GameManager.request_serve(item_key, craft_style_data)`。

### 酒桶

- 材料要进入桶口，且入桶速度达到阈值才接收。
- 多材料在短窗口内累积后匹配配方。
- 抓住酒桶左右摇动，`BrewShakeMeter` 统计方向翻转。
- 摇够后从 `OutputAnchor` 弹出成品，品质为 `normal` 或 `good`。

### 烤架

- 抓着可烤材料按在 `SearZone` 内时，累积单一熟度。
- 离开烤区时定稿：生则保持原样，熟则变成配方成品，过火则变焦糊。
- 肉用 `MeatDoneness` 维护单一熟度；两半视觉共享同一个熟度颜色。

### 炖锅

- 材料进入锅的 `Intake` 后被消耗进内部状态。
- 勺尖在锅内移动会累积搅拌进度。
- 达到 `required_stir` 后按当前材料匹配配方并从 `OutputAnchor` 产出。

## 八、L3 动作风格

当前 MVP 只按上菜速度分类：

| 风格 | 条件 | 默认阈值来源 |
|------|------|--------------|
| 温柔 | 速度低于 gentle 阈值 | `data/craft_style_thresholds.json` |
| 平静 | 中间档 | 同上 |
| 粗鲁 | 速度高于 rough 阈值 | 同上 |

`BarWorkspace._try_serve()` 传入：

```gdscript
{
    "serve_drop_speed": speed,
    "quality": item.quality
}
```

`GameManager` 将其路由给客人结算和 `NarrativeManager.resolve_serve_style()`。当前已接入 Mira Day 4 / wine 的竖切片。

## 十、对话叙事

重要 NPC 对话拆成：

- `{npc_id}_day{N}.pre.dialogue`：上菜前点单/叙事。
- `{npc_id}_day{N}.post.dialogue`：上菜后反馈，读取 `serve_result`、L2 记忆和 L3 风格变量。

当前配置包含 Ryan Day 1-3、Mira Day 4/12。Mira Day 4 的 wine 配方通过 `memory_for` 与 L3 风格联动。

## 十二、开发禁忌

- 不修改 `addons/`，除非明确升级依赖。
- 不把新 `.gd` 或 `.tscn` 散在根目录。
- 新系统优先通过 `GameManager` 路由，不让 `scripts/systems/` 直接互相硬引用。
- 新 Tavern 交互优先接入 `BarWorkspace`、`DeskItem`、`KitchenContainer`、`Brewery`，不要继续扩展旧的固定槽位合成流程。
