## 专项资料

| 路径 | 状态 |
|---|---|
| `docs/ui/` | UI 专项记录，仍可参考；如和当前真源冲突，以当前真源为准 |
| `docs/art/` | 美术专项报告和 contact sheet，仍可参考；图像文件不作为设计规则 |
| `docs/superpowers/plans/` | 历史实施计划，不是当前设计真源 |
| `docs/superpowers/specs/` | 历史规格，不是当前设计真源 |
| `docs/archive/legacy-numbered/` | 重置前旧编号文档，历史参考 |

## 使用规则

- 做功能前先读 `README.md`、`docs/01_开发规范.md`、`docs/02_项目速查.md`。
- 做玩法/系统判断时读 `docs/03_系统总览.md` 和 `docs/04_玩法与内容现状.md`。
- 做 UI/美术/角色资产时读 `docs/05_美术与资产管线.md`，并检查对应 exporter/test。
- 做 UI 屏幕、HUD、菜单、商店、设置、对话或像素 UI 资产时，还必须读 `docs/pixel-ui/` 下的六份文件。
- 不要从 `docs/superpowers/*` 或 `docs/archive/*` 直接推导当前需求；这些文件只能解释历史原因。

## 维护规则

当代码变更改变玩家可见流程、数据结构、测试入口或资产管线时，同一批改动应更新对应真源文档。文档更新要写“当前事实”，不要保留已废弃方案作为默认路径。
