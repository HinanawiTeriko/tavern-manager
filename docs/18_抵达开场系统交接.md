# 抵达开场（Arrival Intro）系统交接文档

> 给接手者（codex 等）：本文档自包含，假设你对本特性零上下文。读完即可独立接手"填美术/音频 + 调参 + 补功能"。
> 配套设计/计划：`docs/superpowers/specs/2026-06-04-arrival-intro-design.md`、`docs/superpowers/plans/2026-06-04-arrival-intro.md`（specs/plans 目录 gitignored，但文件在本地磁盘上，可直接读）。

## 0. 一句话

新游戏点「新游戏」后，先播一段**可跳过的过场动画**（旁白文字淡入淡出 + 可选 L1 Ken Burns 镜头），播完/跳过再进白天地图 DayMap；DayMap 用 match-cut 把相机从贴紧酒馆节点拉开再亮相地点。**当前是"骨架 + L1 镜头"版，零美术零音频即可跑通**；美术/音频是预留槽位。已实现并通过 headless 逻辑测试。分支 `feat/seasoning-system`，提交范围 `293d088..`（约 12 个提交，commit 信息均以 `抵达开场/IntroSequence/match-cut/L1 镜头` 等关键词标注）。

## 2. 端到端流程（数据怎么流）

```
TitleScreen「新游戏」
  → GameManager.new_game(): _pending_intro_handoff = true; call_deferred 切 IntroSequence.tscn
  → IntroSequence._ready(): load_intro(intro.json) → _preload_backgrounds() → _play()
      _play(): 逐 beat 链式 Tween：设背景+镜头起点 → 设文字 → 旁白淡入 →
               (有背景图则 L1 镜头并行推拉，否则 hold) → 旁白淡出
      跳过(点击/按键) 或 播完 → _exit_to_daymap()（单出口，_exited 防重入）→ 切 DayMap.tscn
  → DayMap.show_day(): gm.consume_intro_handoff()==true →
      _play_intro_handoff(): 相机贴酒馆(zoom=MAX_ZOOM) → await fly_to(HOME_POS, 1.0, 1.2) 拉开
        → _refresh_map() 亮相地点 → _maybe_trigger_gather_tutorial() 采集教程
  （继续游戏 / 次日：handoff=false，走 else 分支，相机直接落位，无过场无拉镜）
```

## 3. `data/intro.json` 数据格式（你主要在这里干活）

```jsonc
{
  "bgm": "",                 // 顶层：开场 BGM 的 res:// 路径。空=不放。⚠ 见 §5.A1：当前"填了也不会响"，需先接线
  "beats": [                 // 一拍一个对象，按序播放
    {
      "text": "……",         // 旁白（剧本作者填）。空串=无字
      "bg": "",              // 背景图 res:// 路径。空=走暖色渐变底（Background 隐藏）
      "fade_in": 1.2,        // 旁白淡入秒
      "hold": 2.2,           // 停留秒
      "camera": null,        // L1 镜头段，见下；null=不动
      "sfx": ""              // 本拍音效路径/键。⚠ 同样需接线，见 §5.A1
    }
  ]
}
```

**camera 段（L1 Ken Burns，仅当本拍 `bg` 图成功载入时才生效）：**
```jsonc
"camera": {
  "from": { "zoom": 1.0,  "offset": [0, 0]   },  // 起始：背景 scale=zoom，position=屏幕中心(640,360)+offset
  "to":   { "zoom": 1.12, "offset": [0, -30] }   // 终点：在本拍 fade_in+hold 时长内缓动到此（SINE/EASE_IN_OUT）
}
```
- `zoom`>1 = 放大（推近）；`offset` = 相对屏幕中心的像素位移（平移）。
- **关键约束**：`camera` 只在 `bg` 非空且该图能 `ResourceLoader.exists` 时启用（`intro_sequence.gd:63` 的 `has_cam`）。`bg:""` 时 camera 段被忽略——所以现在第 5 拍虽有 camera 但 bg 空，镜头是哑的，属占位。

**改动安全性**：加/删拍、改文字时长、填 `bg`/`camera`，**都只动 JSON，无需改码**。坏 JSON / 缺文件会优雅降级为"无开场直接进 DayMap"（`load_intro` 多重兜底）。

## 5. 待办清单（"填资源"之外还要干的）

### A. 必须写代码的功能缺口

**A1. 音频播放接线（最大的一块，"填了不会自己响"）**
现状：`intro.json` 有 `bgm`/`sfx` 槽，`load_intro` 也把 `bgm` 读出来了，但 **`IntroSequence` 根本没碰音频**——只驱动文字和镜头。所以填路径 ≠ 出声，得写播放代码。

项目音频系统：`scripts/systems/audio_manager.gd`（`AudioManager`，autoload 经 `GameManager.play_audio_event(key)` 调用，见 `game_manager.gd:548`）。注意两点：
1. 它是**事件键制 + 一次性播放器**：`EVENT_PATHS` 字典里登记 `key→wav 路径`，`play_event(key)` 起一个 `AudioStreamPlayer` 播完自销。
2. **没有循环/BGM 支持**，且 headless 下 `play_event` 直接返回 true 不出声（测试友好）。

建议接法：
- **SFX（每拍）**：把每拍要用的音效在 `AudioManager.EVENT_PATHS` 里登记成 key，然后在 `intro_sequence.gd._play()` 的换拍回调里调 `GameManager.play_audio_event(sfx_key)`。即把 `intro.json` 的 `sfx` 字段当**事件键**而非裸路径（与项目其它音效一致）。
- **BGM（循环）**：AudioManager 不支持循环，别硬塞。最简单：在 `IntroSequence.tscn` 加一个独立 `AudioStreamPlayer` 节点，`intro_sequence.gd._ready()` 里若 `bgm` 非空就 `load(bgm)`、设 `stream.loop=true`、`play()`；`_exit_to_daymap()` 里淡出/停掉。注意 headless 守卫：`if DisplayServer.get_name()=="headless": return`，避免测试环境报错。

**A2. 末拍与 DayMap 的视觉对齐（match-cut 收尾）**
现在是占位渐变底，看不出接缝。等酒馆外景/内景图到位后，要让**开场最后一帧（酒馆）的构图**和 DayMap 起始画面（相机贴 `__home__` 节点、`HOME_POS=(1000,1080)`、起始 `zoom=MAX_ZOOM=1.6`）对得上，拉镜才"无缝"。可能要调末拍 `bg` 的取景/`camera.to`，以及 `day_map_view.gd:148` 的 `fly_to(HOME_POS, 1.0, 1.2)` 起止参数。

### C. 可选后续（明确推迟过，不影响完整性）

- **L2 多层视差**：想要更强纵深时，把背景切前/中/远景多层 PNG（用 §6 像素化管线），镜头移动时各层不同速度。需扩 `IntroSequence.tscn` 多 Sprite + `_play` 里分层 tween。
- **边界小补**：`_apply_background()` 的 else 分支没重置 `_background.scale/position`。只有"某拍有图无 camera 且紧跟在一个 camera 拍之后"才会继承上一拍镜头姿态——当前数据无此排列；若你排美术拍时出现，在 else 分支补一行重置即可。

## 6. 美术 / 音频资源：放哪、怎么产

- **美术**：3–5 张开场图（建议：地下城全景 → 酒馆外景夜 → 推门内景）。放 `assets/textures/`（参考现有 `assets/textures/ui/` 结构），把 `res://` 路径填进对应拍的 `bg`。产图走项目既有像素化管线（见 `docs/07_美术需求文档.md` 及记忆中的"UI美术像素化工作流"：GPT 手绘稿 → PIL 抠白底+像素化 → 离屏 1280×720 验证；BGM 提示词见 `docs/12_BGM提示词_Suno.md`）。末拍建议落在"酒馆"以服务 match-cut。

## 7. 坑与硬约束（务必遵守，本项目踩过）

1. **`.gd` 与 `.tscn` 不同目录**：脚本 `scripts/<子目录>/`，场景 `scenes/<子目录>/`。
2. **所有 `load`/`preload` 用 `res://` 绝对路径。**
3. **`.uid` 文件要提交**（本项目跟踪 151 个 .gd.uid；新脚本的 .uid 已提交）。但**绝不提交 `.godot/`**；git 显式 `git add <file>`，**不要 `git add .`/`-A`**。commit 信息用 `type: 简述`。
4. **零美术降级是特性不是 bug**：所有 `bg:""` 走渐变底，camera 哑火属正常。
5. **GDScript 闭包按值捕获**：`_play()` 里 `for beat` 内的 lambda 捕获 `beat` 是按值快照、每拍独立，正确（不是 Python/JS 那种共享循环变量的坑），别"修"。
6. **L1 镜头时序已修**：camera tween 用 `parallel()` 挂在旁白淡入锚点上，与文字并行；camera 拍的并行组时长 `dur=fade_in+hold` 同时承担 hold，故 camera 分支**不再单独 `tween_interval`**（非 camera 拍才 hold）。改 `_play` 时别破坏这个分支结构。
7. **预存测试失败（与本特性无关，别误查）**：`scenes/test/test_day_map_system.tscn` 有 **1/40 失败** `evidence not in story bag before reading`。根因是 `game_manager.gd.grant_mine_document()` 授予故事物品时就入背包，与该测试断言矛盾，属 Ryan 切片/库存遗留，**5ff0eb4 起就存在，不是抵达开场引入的**。详见仓库 `feat/seasoning-system` 历史。
8. **headless 测不了场景视觉**：逻辑测试只覆盖 `load_intro` 解析 + handoff 一次性语义；镜头/跳过/拉镜的观感必须 GUI 走查。

## 8. 关键机制备忘（改动前先懂）

- **一次性 handoff**：`GameManager._pending_intro_handoff` 只在 `new_game()` 置 true，`consume_intro_handoff()` 读后即清。所以 match-cut 只在新游戏首进 DayMap 跑一次；continue/次日恒 false 走 else 直接落位。
- **单出口**：跳过和播完都经 `_exit_to_daymap()`，`_exited` 防重入并 kill timeline 再切场景。改播放逻辑时保持这个单出口。
- **教程时序**：采集教程已从 `show_day` 内联挪到 `_maybe_trigger_gather_tutorial()`，在 match-cut 拉镜+reveal **之后**才触发（避免盖在亮相前的空地图上）。两个分支都在 reveal 后调它。
