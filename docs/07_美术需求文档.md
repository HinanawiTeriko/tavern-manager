## 快速参考卡（给美术）

- **游戏**: 2D 像素涂鸦模拟经营，地牢酒馆老板视角
- **分辨率**: 1280×720，背景画布 640×360 绘制后 2x 最近邻缩放
- **风格**: 简陋像素涂鸦（MS Paint 手绘感，纯色块 + 1px 黑轮廓），整体统一即为风格
- **色板**: 全游戏统一 24 色，纯色不渐变
- **角色规格**: 48×64 px 绘制，3x 最近邻缩放后显示为 144×192
- **图标规格**: 16×16 px 绘制，3x 缩放后 48×48；小图标 8×8 px 绘制
- **文件格式**: PNG，所有素材需支持透明背景（背景/BG 除外）
- **命名**: 全小写 + 下划线，如 `tavern_bg.png` / `ryan_smile.png`
- **交付**: 按 P0 → P1 → P2 分批，每批一个 zip

## 1. 美术风格总纲

### 1.1 核心关键词

`简陋像素涂鸦` `纯色块` `1px 黑轮廓` `MS Paint质感` `24 色板` `刻意简陋`

### 1.2 通用规则（所有资产遵守）

| 规则 | 说明 |
|------|------|
| **像素** | 所有资产以目标像素尺寸直接绘制，非滤镜降质。在 Godot 中以 2x/3x 最近邻（nearest-neighbor）缩放显示 |
| **轮廓** | 可交互物体（角色/物品/按钮/工具）一律用 1px 纯黑 `#000000` 描边。背景元素不需要描边 |
| **纯色** | 无渐变、无半透明、无抖动 dithering。每个形状只用单一色块填充 |
| **色板** | 全游戏严格 24 色，从本文档第 3 节色板取色，不引入新颜色 |
| **光照** | 不画光照和阴影渐变。需要明暗区别时，用纯色块切换（亮面浅色、暗面深色），最多 2-3 个色阶 |
| **细节** | 克制细节，每个物体只画出核心辨识特征。多一笔不如少一笔 |

### 1.3 参考作品

| 作品 | 参考点 |
|------|--------|
| **Downwell** | 极限 3 色板 + 小像素角色 + 纯色块打击感 |
| **Minit** | 1-bit 黑白极简，正负空间设计 |
| **Reigns** | 简约像素角色 + 大块纯色 UI |
| **早期 Flash 涂鸦游戏** | MS Paint 式的粗粝手绘感，线条不追求完美 |

### 1.4 与 v3.0 (暗黑卡通) 的主要区别

| 维度 | v3.0 暗黑卡通 | v4.0 像素涂鸦 |
|------|-----------|--------------|
| 绘制方式 | 手绘描边 + 赛璐珞填色 | 像素点直接绘制 |
| 线条 | 1-3px 粗细变化，笔触感 | 统一 1px 纯黑轮廓 |
| 颜色 | 平滑渐变 + 柔光 | 纯色，无渐变 |
| 阴影 | 暖褐渐变 | 纯色色阶（最多 2-3 层） |
| 缩放 | 直接目标尺寸 | 2x/3x nearest-neighbor |
| 工期 | 长，需要手绘功力和画师 | 短，画图软件几分钟可出图 |
| 动效 | 帧动画为主 | 2-3 帧简陋摆动（保留同样弹性 UI |
| 字体 | 手写体 + 衬线 | 像素点阵字

## 2. 动效规范

### 2.1 按钮交互动效

按钮需要以下状态贴图支持交互动效：

| 状态 | 触发 | 动效描述 | 时长 |
|------|------|----------|------|
| **Normal** | 默认 | 按钮本色，无描边 | — |
| **Hover 进入** | 鼠标移入 / 选中 | 按钮轻微抖动一次 + 切换为 hover 底色 + 出现白色边框 | 抖动 0.15s |
| **Hover 离开** | 鼠标移出 | 按钮再抖动一次 + 回到 normal 底色，白色边框消失 | 抖动 0.15s |
| **Pressed** | 按下 | 颜色回到 normal 底色（下沉感） | 持续按住 |
| **Released** | 松开（仍在按钮上方） | 回到 hover 底色 + 白色边框 | 即时 |
| **Disabled** | 不可用 | 去饱和/变灰，无 hover 反馈 | — |

### 2.2 弹性动效（待细化）

以下交互动效方向已确定，具体弹性参数待后续迭代：

- 面板/弹窗出现时的入场过渡
- 合成成品出现时的视觉反馈
- 数值变化（金币/声望）的跳动过渡
- 角色表情切换的过渡方式
- 口味标签/状态提示的出现和消失

### 2.3 动效交付物

- 按钮 Hover 态贴图（白色边框版本）：`btn_wide_hover.png` / `btn_small_hover.png`
- 按钮 Pressed 态贴图（回到 normal 色）：`btn_wide_pressed.png` / `btn_small_pressed.png`
- 动效的具体实现参数由代码侧通过 Tween/AnimationPlayer 控制，美术侧交付状态贴图即可
