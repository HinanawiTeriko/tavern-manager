# 开发规范

## 基本环境

- Godot 版本：4.6.3 standard。
- 主语言：GDScript。
- 主场景：`res://scenes/ui/TitleScreen.tscn`。
- 分辨率基准：1280x720，`window/stretch/mode="canvas_items"`。

## 合同优先

这些对象被脚本、测试或 autoload 视为外部合同，不能随意改名、移动或改签名：

- Autoload：`GameManager`、`DialogueManager`、`TutorialManager`。
- 主要场景：`TitleScreen`、`DayMap`、`Tavern`、`LedgerScreen`、`EndingScreen`。
- 高频 UI 节点：`CustomerArea/CustomerSprite`、`BarWorkspace`、`CustomerDropArea`、`UILayer/ResultPanel`、`UILayer/GatheringToast`。
- 公开方法：`GameManager.register_view()`、`visit_day_location()`、`request_serve()`、`restart_current_day()`、`get_today_rumors()`、`TavernView.show_customer()`、`show_customer_reaction()`。

如果必须破坏合同，先说明原因、影响范围和替代方案，再改测试和实现。

## 工作方式

1. 先读相关代码和测试，不按旧文档猜。
2. 对 bug 先找根因，再改。
3. 对行为变化先补测试，再实现。
4. 改 UI 场景前先列计划触碰的 `.tscn` 和脚本。
5. 保持改动边界小：数据改数据，UI 改 UI，系统逻辑改系统。
6. 不回退未确认的本地改动。

## 数据优先

能配置到 JSON 的内容优先放在 `data/`：

- 物品、配方、价格：`items.json`、`recipes.json`
- 地点、奖励、贴文：`locations.json`
- 传闻：`rumors.json`
- 常客：`regular_customers.json`
- 客群：`guest_group_profiles.json`
- 胃口标签：`guest_appetites.json`、`food_attributes.json`
- 推理题：`inference_puzzles.json`
- NPC：`npcs.json`

脚本负责解释数据，不把可调内容硬编码进 UI。

## 测试

优先跑最小相关测试，再补高价值回归。Godot 测试格式：

```powershell
& 'D:\Godot_v4.6.3-stable_win64.exe\Godot_v4.6.3-stable_win64_console.exe' --headless --path 'D:\game\tavern-manager' 'res://scenes/test/test_rumor_appetite_system.tscn'
```

常用组合：

- DayMap：`test_day_map_system.tscn`、`test_day_map_scrollbars.tscn`
- Tavern：`test_tavern_patience_ui.tscn`、`test_workspace_scene_recovery.tscn`
- 传闻/菜单/常客：`test_rumor_appetite_system.tscn`、`test_regular_customers.tscn`
- 剧情：`test_ryan_slice_system.tscn`、`test_mira_toby_route_matrix.tscn`、`test_evelyn_grey_ledger_line.tscn`
- 结算/重开：`test_night_settlement_screen.tscn`、`test_restart_current_day.tscn`
- 存档：`test_save_roundtrip.tscn`、`test_save_system.tscn`
