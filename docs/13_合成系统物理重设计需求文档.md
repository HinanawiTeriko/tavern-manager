## 2. 设计目标

- 合成的载体 = 物理拖拽 + 重力，物品有重量/惯性/堆叠/滚动
- 物理动作本身是叙事表达：力度、角度、节奏成为玩家对客人态度的回应
- 30 天 × 多客人循环下保持新鲜感（渐进解锁容器、可选深度动作）
- **剧情为主，但物理也好玩**——VA-11 Hall-A 式吧台调酒师叙事

---

## 3. 三层叙事框架（核心）

合成系统通过三个独立但有交互的层影响剧情：

| 层 | 输入 | 效果 | 例子 |
|----|------|------|------|
| **L1 物品属性** | 成品属性（辛辣/清香/咸香/安眠） | 决定客人宿命 | 安眠属性触发 Ryan 成瘾结局 |
| **L2 配方** | 玩家做了什么具体酒/食物 | 解锁客人背景故事 | 给老兵葡萄酒 → 触发战争往事 |
| **L3 动作风格** | 物理操作细节（力度/角度/节奏） | 信任阀门：决定 L2 故事是否被讲出 | 慢倾倒葡萄酒 → 老兵开口；砸一下 → 喝完不说话 |

**三层关系**：
- L2 是"想讲"，L3 是"愿讲"，两者都对，故事才被讲出
- L1 宿命独立运作
- 路人客人只读 L1，命名 NPC 读 L1+L2，重要 NPC 走完整三层

**剧本作者负担分级**：

| NPC 类型 | 写作量 |
|---------|--------|
| 路人 / Day 1 type | 1 段通用反馈，不读 L3 |
| 命名 NPC | 2-3 段（preferred / disliked / 其他）|
| 重要 NPC（Ryan、老兵 等） | 5 段（4 corners + neutral）× 多天 |

## 4. 物理基础（基于 gravity_test）

| 项 | 说明 |
|----|------|
| 物品本体 | RigidBody2D + 自由旋转 + 物理材质（bounce 0.25, friction 0.6） |
| 拖拽 | PinJoint2D 钉在按下时的点击位置（不是物品中心） |
| 力度读取 | 松手时的 `linear_velocity` |
| 倾斜读取 | `body.rotation` |
| 关键涌现 | 抓物品的不同部位决定倾斜姿态（抓瓶颈 = 自然倾倒姿态） |

## 5. 吧台工作面

### 5.1 布局（Day 1）

```
┌──────────────────────────────────────────────┐
│  TopPanel（金币/声望/天数）                   │
├──────────────────────────────────────────────┤
│            客人区（立绘 + 订单 + 耐心条）      │
├──────────────────────────────────────────────┤
│  ┌──────┐    ┌──────┐    ┌──────┐             │
│  │ 酒桶 │    │ 炖锅 │    │ 烤架 │             │
│  └──────┘    └──────┘    └──────┘             │
│      （吧台自由放置区，物品可自由摆放堆叠）    │
├──────────────────────────────────────────────┤
│  快捷栏 0-9（材料源头入口）                    │
└──────────────────────────────────────────────┘
```

### 5.2 规则

- 物品从快捷栏拖出后即变成物理体，落在桌面任意空位
- 物品互相有碰撞体积，可叠加
- 容器固定不可移动（StaticBody2D）
- 桌面物品掉出屏幕外自动销毁
- 桌面清理：保留现有 ClearBtn 一键回收

## 6. 容器系统（"容器 = 操作"）

### 6.1 Day 1 起手 3 站

| 容器 | 等价动词 | 示例配方 |
|------|---------|---------|
| **酒桶** (`barrel`) | 酿 | 麦芽 → 麦芽酒；葡萄 → 葡萄酒 |
| **炖锅** (`pot`) | 炖 | 麦芽+葡萄 → 烩饭；生肉+麦芽 → 肉汤 |
| **烤架** (`grill`) | 烤 | 生肉 → 烤肉；面粉 → 面包 |

### 6.2 后续解锁

| 天数 | 解锁 | 新动词 | 触发剧情 |
|------|------|--------|---------|
| Day 5 | 砧板 | 切 | 流浪汉送旧砧板 |
| Day 10 | 冰桶 | 冰镇 | 雪山旅人留下冰桶 |
| Day 15 | 调味碾钵 | 研磨 | 草药商人赠予 |

**5 站封顶**。每次解锁伴随剧情事件——机制扩展 = 叙事推进。

### 6.3 可选用心动作（提升品质）

每个容器有一个**可选**的物理动作。新手不学也能玩，老玩家挖深度。

| 容器 | 用心动作 | 物理实现 |
|------|---------|---------|
| 酒桶 | 摇晃 | 抓住桶左右拖拽 |
| 烤架 | 翻面 | 抓起物品转 180° |
| 锅 | 搅动 | 用"勺子"物理实体在锅里画圆 |
| 砧板 | 连续切 | 连续点击 |
| 冰桶 | 晃冰 | 抓住容器晃动 |

## 7. 小循环 timeline

```
0s    客人进门
~5s   听单（叙事段）
~10s  取材
~14s  ★ peak 1：扔进容器（力度判定）
~17s  ★ peak 2：[可选] 用心动作
~22s  ★ peak 3：倾倒进杯（精细度判定）
~27s  ★ peak 4：上菜（递送方式）
~33s  客人反馈
```

- 总长 30-40s，60s 耐心留 ~25s 容错
- 4 个 juicy peak 分散在循环里
- 重要 NPC 讲故事时暂停耐心计时（GuestSystem 加 `is_narrative_paused`）

## 8. 配方模型

### 8.1 数据格式

```json
"ale": {
    "name": "麦芽酒",
    "ingredients": ["麦芽"],
    "container": "barrel",
    "attribute": "",
    "memory_for": {
        "veteran": "day5_war_story",
        "merchant": "day12_wife_story"
    }
}
```

### 8.2 配方库规模

- 现有 9 个 → 扩到 15-25 个
- (NPC × 配方) → 故事段：同一配方对不同 NPC 触发不同故事
- 30 天 × 多 NPC 不需要全独有配方

## 9. L3 动作风格分类

### 9.1 5 类标签（2 轴 × 2 级 + 中性）

|  | 匆忙 / 快 | 仔细 / 慢 |
|---|---|---|
| **大力** | **粗鲁** | **果断** |
| **轻柔** | **敷衍** | **温柔** |

中性：**平静**（默认）

### 9.2 物理参数源

```gdscript
craft_style = {
    "max_throw_speed": float,    # 拖拽释放时的最大 linear_velocity
    "max_pour_tilt": float,       # 倾倒时达到的最大角度
    "pour_duration": float,       # 倾倒持续时间
    "serve_drop_speed": float,    # 上菜放下瞬间的速度
    "time_to_serve": float,       # 从客人下单到 serve 的总时长
    "operation_pause_avg": float  # 操作间平均间隔
}
```

阈值放 `data/craft_style_thresholds.json` 可热调。

### 9.3 NPC 偏好

```json
"veteran": {
    "preferred_styles": ["温柔", "果断"],
    "disliked_styles": ["粗鲁"]
}
```

- 匹配 preferred → 信任 +，L2 故事解锁
- 落到 disliked → 信任 -，可能触发负面分支
- 中性档 "平静" → 不动信任值（安全默认）

### 9.4 混合动作的判定规则

玩家可能"扔得粗鲁但倾倒温柔"——**取上菜时的动作风格为主**（最接近递给客人的瞬间），其他动作作为加权调整。

### 9.5 实施策略

**MVP 先做 3 类（粗鲁/平静/温柔，单轴：力度）**，试玩后扩 5 类。所有数据接口（craft_style 字典、NPC 字段）按 5 类设计，分类函数返回 3 类，扩展时只改一个函数。

## 10. 失败/试错处理

核心原则：**软兜底优先**。

| 失败 | 处理 |
|------|------|
| 容器装错材料 | 冒烟标红，材料消耗，容器可清空再用 |
| 扔出桌面 | 销毁，材料消耗 |
| 倒撒了一半 | 仍可上菜，品质降级 |
| 物品掉地上 | 5 秒内可拾回 |
| 客人耐心耗尽 | hard 失败，客人当晚不再来，关系-1 |

只有"耐心耗尽"是 hard 失败。其余可恢复但有代价。

## 13. 上菜动作

默认**递给客人**：
- 拖到客人前桌位 → 松手 → 物品轻放下
- L3 读 `serve_drop_speed`

剧情可触发其他变种：
- 滑过吧台：醉汉/雇佣兵能接住
- 扔给客人：撒泼式，通常负面反馈

## 15. 实施路径

### 15.1 分 3 阶段交付

| 阶段 | 范围 | 工时 | 验收 |
|------|------|------|------|
| **MVP 1** | 1 容器（酒桶） + 基础扔 + 3-5 配方 + 上菜 | 4-5 天 | 完整闭环可测试扔+倾倒+上菜手感 |
| **MVP 2** | 扩到 3 容器 + 15 配方 + 摇晃酒桶 | +3-4 天 | Day 1 完整体验 |
| **MVP 3** | L3 craft_style + 3 类分类 + 3 重要 NPC 接入 | +3-5 天 | 三层叙事咬合完整 |
| **润色** | 翻面/搅动/更多 NPC L3/正式美术 | 持续 | — |
| **总计完整版** | | **~13-15 天** |

每阶段交付都是**可玩状态**——不会出现"做了 10 天但没法玩"的死时间。

### 15.2 非代码硬成本

| 项 | 来源 |
|----|------|
| 容器美术（3-5 个） | 美术（占位可先用色块） |
| 16 个新成品 icon | 美术 |
| 配方命名 / 属性 / 价格设计 | 剧本 + 游戏设计 |
| 阈值精调（内测） | 全员 |
| 重要 NPC 的 L3 分支对话（粗估 ~190 段） | 剧本作者 |

### 15.3 能省工时的诀窍

1. **容器用占位美术先做** —— 色块 + Label 也能验证机制，省美术等待
2. **复用 stir/heat 手势代码** —— `craft_station.gd` 的 `_stir_total_angle` 圆周检测、`_heat_progress` 按住检测可改用到搅动 / 翻面
3. **配方书简化** —— docs/14_legacy 的填空机制不做，省 2-3 天
4. **教学走 TutorialManager** —— 现有 overlay 加几个 step，不写新系统
