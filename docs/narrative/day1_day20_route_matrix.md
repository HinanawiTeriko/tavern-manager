# Day1-Day20 剧情经营路线矩阵

更新时间：2026-06-18

## 目的

这份矩阵只整理当前实现，不改剧情数据。目标是回答一个问题：玩家每天通过哪个系统拿到哪类线索，线索怎样进入推断、物理调查、命运账本和最终结局。

当前融合方向：

- 白天地图不是单独采集小游戏，而是“材料、地点入口、风声预告”的同一层。
- 夜晚经营不是只出材料做菜，而是用菜单吸引对应客群，让普通客人给风声，重要 NPC 给人心。
- 物理调查不是额外解谜，而是把白天入口找到的纸面证据变成可用于推断的证据。
- 推断题不是单独脑筋急转弯，而是把风声、人心、证据合成事实 flag。
- 命运账本不是纯展示，而是玩家判断当前还缺什么、哪条线将在哪里结算的主线索引。

## 线索类型

| 类型 | 含义 | 当前主要来源 |
| --- | --- | --- |
| 风声 | 外部传闻、告示板、普通客人口风 | DayMap 地点传闻、普通顾客、告示板 |
| 人心 | 角色本人暴露的动机、恐惧和关系 | 莱恩、米拉、托比、伊芙琳的重要 NPC 夜谈 |
| 证据 | 能收入账本、背包、物理调查的文件或物件 | 矿道、托比住处、赔付登记处、黑齿账、清算台 |
| 事实 | 推断题把多条线索合成的结论 | CleanTableInferenceScreen |
| 后果 | 角色线或灰账线的不可逆结算 | Day3、Day12、Day20 的命运轨 |

## 当前已经解决的融合点

- 商店不再是单独“听传闻”按钮，商店访问和 DayMap 地点传闻已经合并。
- DayMap 访问地点后的提示会直接写出听到的传闻内容。
- 重要 NPC 不再默认每晚第一个来，除 Day3 命运揭晓外会先经过普通经营。
- 莱恩 Day2 只点烤肉，Day3 佣兵点烤肉，避免菜单缺草药清汤一类错配。
- 普通客人给米拉旧路风声已经按来源组和特质限制，不会由完全无关客群乱泄露。
- 调查场景拿到证据时会告诉玩家新解锁了哪道推断。
- 推断桌空状态会显示下一题缺什么线索，并标出风声、人心、证据来源。

## 仍然割裂的点

1. `back_alley_boy` 和 `one_person_walk` 的数据源还是“夜晚客人”。实际来源是 Day6 托比本人，应该改成“托比夜谈”或“瘦小少年”。
2. Day7-Day11 的米拉旧路风声依赖普通客群和菜单，但命运账本没有清楚告诉玩家“今晚要吸引旧路熟人/贸易客继续问旧话”。
3. 米拉信任阈值影响 Day12 结局，但 UI/账本没有提前告诉玩家“递委托不等于她会回头”。
4. 莱恩药酒路线能救人，但后续只轻描淡写地记录“酒中有沉睡花粉”，缺少和伊芙琳线的压力连接。
5. Day14-Day17 灰账线结构是地点取初证、清算台压关键证，但普通提示仍容易让玩家以为“点过地点就完事”。
6. Day18-Day19 是公开账补全期，但目前主要依赖推断桌缺口提示，命运账本和日间传闻还不够主动。
7. Day21-Day30 目前主要是普通经营循环，Day20 后主线已结，需要单独规划后日谈或长期经营目标，不应该混在 Day1-Day20 主线修复里。

## 完成标准

这一阶段算完成，不是因为剧情全修完，而是因为 Day1-Day20 的主线链路已经被拆成可执行切片：

- 能说清每条线索来自哪里。
- 能说清每个地点、夜晚客人、调查场景、推断题在主线里承担什么职责。
- 能说清三条人物线怎样汇入 Day20 灰账结局。
- 后续改动可以按小切片提交，不需要一次性重写剧情、经营、调查和 UI。
