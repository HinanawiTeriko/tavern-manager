# 托比的落脚处 · 物理调查英雄场景 — 设计

> 日期：2026-06-08 ｜ 分支：feat/blacksouls-intro（米拉/托比线在此）
> 承接 [物理调查英雄场景方向](2026-06-02-physical-investigation-scene-mine-pilot-design.md) 与已实装的废弃矿道试点。

## 1. 背景与目标

米拉线（抛下学徒托比，四结局）的逻辑层已实现并 headless 测试绿，但米拉/托比目前只有轻量「点访问→文字弹窗」式的 DayMap 地点，没有像莱恩矿道那样的**物理调查英雄场景**。

目标：给**米拉/托比这桩事**一个专属的物理调查场景——玩家亲手挖出那份险委托。

**归属单位 = 事件，不是人头。** 莱恩的矿道覆盖他那桩事（血斧陷阱）；米拉与托比是同一桩事（托比为钱接的黑齿矿脉护送委托、米拉当年抛下他），故**共用一个场景**，不各做一个。这与既定的「少而精的英雄场景」原则一致：每桩核心事件一个手摆场景，而非每个节点都套模板。

## 2. 场景与叙事载荷

- **地点**：托比的落脚处（现有 DayMap 地点 `toby_lodging`，anchor `toby_lodging_lane_01`，market 区，day≥6）。当前它只是纯伏笔点访问，本设计将它**升级为物理调查英雄场景**。
- **载荷**：这场调查是玩家拿到 `toby_contract` 真相的**新方式，物理前端取代现有的文字弹窗**。守「单一真相源」。
- **情感处境**：你在翻一个被抛下、绝望少年的私人窝。与矿道（灾**后**现场、人已死、冷·法证）相对——这里是灾**前**、孩子还没上路、你攥着他还能改的命（热·愧·私密）。

## 3. 与莱恩矿道的区别（防换皮）

公共基类只是管道；区别活在管道之上。

| 轴 | 莱恩·矿道（已实装） | 托比·落脚处（本设计） |
|----|---------------------|----------------------|
| 物理动词 | 扒开矿道碎石/废墟——挖掘、减法揭示 | 翻一个活人的窝：掀褥子、倾倒、撬小盒——搜检 |
| 时态/处境 | 灾后，人已死，证据是「已发生的陷阱」 | 灾前，孩子还没上路，你攥着他「还能改的命」 |
| 情感寄存器 | 冷·法证·恐惧（尸体旁的染血纸） | 热·愧·私密（闯入绝望孩子的私人空间） |
| 检视→观察 | 法证式："这是个陷阱" | 肖像式："这孩子得多绝望才接这单" |
| 专属机制 | （无） | **拼合撕碎的委托书** |

### 专属机制：拼合撕碎委托书

玩家找到的不是完整委托书，而是**被托比亲手撕碎的几片**——他动摇过、撕了，却没舍得扔。玩家要把碎片**拖到一起、对齐吸附拼合**，拼全了才授予 `toby_contract`。

- **主题咬合**：你拼回这张纸 = 替这个还没下定决心的孩子做决定。物理动作直接是叙事。
- **可辨识**：碎片对齐吸附是矿道（扒碎石）里没有的动词，一眼不同。
- **成本**：在现有拖拽+重力上加一层「碎片对齐吸附」，约 1 块新积木（在既定「2 块新积木」预算内）。

## 4. 场景内容（深浅分层，沿用矿道）

- **浅层（进场可见、免费）**：窝里的破烂——快灭的油灯、半块干硬面包、不合身的旧外套、也许一件米拉早年给的小物件。检视各给**一句肖像式观察**，堆出"这孩子怎么活"。**不收任何东西**（守北极星：调查收线索不收材料）。
- **深层（gating 线索）**：撕碎的委托书碎片，散在不同角落（褥子下、灯影里、扣在碗底）。玩家先翻找出碎片，再拖到一起对齐拼合，拼全才授予 `toby_contract`。

## 5. 可错过 + 连锁（复用已实现的结局逻辑，不动它）

- 落脚处是 cost-1 DayMap 地点（day≥6）；不花体力去就拿不到 → **可错过**。
- 连锁（`narrative_manager` 已实现）：没拼出 `toby_contract` → 无法告知米拉 → `told_mira_truth` 恒 false → 「撬动米拉担责」路线关闭 → 只剩掮客 40 金兜底 或 托比死。**调查场景只换"真相入口"的物理前端，结局判定一行不改。**
- 离场若没拼全，给**软提示**（如"还有几片纸散在角落"）防闷亏——矿道已有离场软提示模式可复用。
- 公会板 `toby_commission` 贴文改为**只描述+指路**（"托比接了份高价险委托"，指向落脚处），**不再授予文档**。

## 6. 架构

英雄场景 #2 是把可复用积木从矿道试点里抽出来的时机。

- 抽公共基类 **`InvestigationScene`**（暂名，管道层）：`DragController` 钉子拖拽、占位 RigidBody 线索物、抓起线索物→`GameManager.grant_*_document`、进场把 DocumentOverlay reparent 到高层 CanvasLayer、离场软提示——这些矿道趟过的坑都归基类。
- **矿道回迁**到新基类（小重构，回归测试兜底）。
- **托比落脚处**作为手摆的独立场景继承基类，叠加**拼合机制**与场景专属内容。
- 守「复用积木不复用房间」：共享管道，剧本质感在手摆层。

## 7. 测试（headless 能验的）

- `toby_contract` 授予的幂等/时机（仿 `test_mine_investigation`）。
- 改 `test_daymap_postings`：断言公会板贴文**不再**授予 `toby_contract`。
- 拼合数据层：碎片集齐标志 → 才授予（物理对齐手感只能编辑器走查）。
- 回归：`test_mira_line` 仍绿（结局逻辑没动）；矿道回迁后 `test_mine_investigation` 仍绿。
- 物理手感（翻找、对齐吸附、拼合）须人工在编辑器走查。

## 8. 范围与非目标

- **范围**：仅托比落脚处一个场景 + 公共基类抽取 + 矿道回迁 + 上述测试与数据改动。
- **非目标**：不为其他角色新建调查场景（本设计同时确立"每桩核心事件一个英雄调查场景"的 pattern，后续角色各自走 spec→plan→code）；不改米拉线结局判定逻辑；不改 30 天循环 / `LAST_DAY` 走查脚手架（那是合入 main 前的独立决策）。
