# 容器右键退料、基础设置菜单与局内主体美术冻结设计

> 日期：2026-06-01
> 状态：已确认，待实现计划
> 范围：替换清洗盆交互，新增基础设置菜单，明确当前可推进的局内美术资产

## 1. 背景

当前 Tavern 物理工作台使用清洗盆清空酒桶或炖锅中的未结算材料。玩家需要把容器拖到右侧清洗盆并停留 `0.8s`。该交互占据桌面空间，也不适合当前较小的容器尺寸。

标题页当前提供“重新开始当前日”。该入口价值较低，项目尚无正常游戏常见的显示和音量设置。

局内美术需要开始推进，但完整酒馆背景和工具固定位置仍可能受后续功能影响。应先冻结可独立替换的主体资产和程序锚点，避免把工具画死在背景中。

## 2. 目标

- 删除清洗盆，以右键容器逐份退料替代。
- 标题页和局内菜单共用一套基础设置面板。
- 支持窗口化、全屏、分辨率和主音量设置，并跨启动保存。
- 冻结容器、工具和当前内容资产的接入边界，使局内主体美术可以立即开始。
- 统一标题页、设置面板和局内菜单的视觉语言，复用标题界面已经确定的像素风格。

## 3. 非目标

- 不保留“重新开始当前日”标题页入口。
- 不实现音乐、音效分轨音量。
- 不实现语言、键位、垂直同步或高级图形设置。
- 不完成酒馆背景最终稿，不把工具位置画死在背景中。
- 不在本轮完成局内 HUD、耐心条、快捷栏和对话框的完整皮肤。
- 不修改烤架现有材料拿取方式。

## 4. 容器右键退料

### 4.1 玩家交互

- 玩家右键命中酒桶或炖锅时，容器弹出最后投入的一份材料。
- 弹出的材料从容器 `OutputAnchor` 附近向上蹦出，重新成为桌面上的 `DeskItem` 物理实体。
- 连续右键按后进先出顺序逐份取回材料。
- 空容器右键不产生效果。
- 烤架上的材料保持现状，玩家直接拿走桌面物理实体。

### 4.2 状态规则

- 酒桶退料后重置摇晃进度，保留尚未退出的其它材料。
- 炖锅退料后重置搅拌进度，保留尚未退出的其它材料。
- 退料不直接返回背包，不改变库存计数。材料重新成为桌面实体后，沿用现有越界恢复规则。
- 成品不可通过右键容器退回。

### 4.3 代码边界

- 酒桶和炖锅各自暴露同义接口，例如 `pop_last_ingredient() -> String`。
- `BarWorkspace` 负责右键命中测试、调用容器接口和生成弹出物理实体。
- 容器只维护内部材料状态，不直接引用 `BarWorkspace`。
- 使用已有 `OutputAnchor` 作为弹出锚点，避免另建重复位置来源。

### 4.4 清洗盆移除范围

- 删除 `Tavern.tscn` 中的 `WashBasin` 节点和碰撞形状。
- 删除 `BarWorkspace` 中清洗盆引用、停留计时、清空调用和勺子浸没判定。
- 删除或改写清洗盆相关教程、测试、占位音效和占位 VFX 约束。
- 保留“整理桌面”：它继续负责散落物品恢复和容器、工具归位，不清空容器内部状态。

## 5. 基础设置菜单

### 5.1 入口

- 标题页用“设置”替换“重新开始当前日”。
- Tavern 局内菜单增加“设置”入口。
- 两个入口打开同一套设置面板和同一套设置逻辑。

### 5.2 首版选项

| 选项 | 可选值 | 行为 |
|---|---|---|
| 显示模式 | 窗口化、全屏 | 即时切换 |
| 分辨率 | `1280x720`、`1600x900`、`1920x1080` | 窗口化时即时调整窗口尺寸 |
| 主音量 | `0-100%` | 即时调整 Master 总线 |

全屏时保留玩家选择的窗口分辨率。切回窗口化后恢复该尺寸。

### 5.3 持久化和系统边界

- 新增 `SettingsManager`，作为独立系统管理默认值、加载、应用和保存。
- 使用 `ConfigFile` 写入 `user://settings.cfg`。
- 设置与剧情存档分离。开始新游戏、继续游戏和清除剧情存档都不应覆盖玩家设置。
- `GameManager` 持有并暴露 `SettingsManager`，UI 不直接维护设置真相源。
- 启动后尽早加载并应用设置，使标题页进入时已使用玩家配置。
- Headless 环境跳过窗口切换，但仍允许读取和验证设置数据。

### 5.4 UI 结构

- 新增可复用设置面板场景和脚本。
- 面板包含标题、显示模式选择、分辨率选择、主音量滑杆和“返回”按钮。
- 设置即时生效，不提供“应用”按钮。
- 标题页打开设置时覆盖标题页菜单；关闭后回到标题页。
- 局内打开设置时覆盖局内菜单；关闭后回到局内菜单。

### 5.5 菜单美术统一范围

菜单美术跟随当前标题界面已经确定的像素视觉语言：深色底、琥珀色重点、高对比文字、像素字体和简洁装饰。标题页、设置面板和局内菜单使用同一套组件，不分别绘制固定排版大图。

本轮统一覆盖：

- 标题页主菜单，包括新增“设置”入口。
- 设置面板。
- Tavern 局内菜单中的配方、背包、教程、整理桌面、关闭和设置入口。

本轮交付为可复用组件：

- 九宫格拉伸弹窗底板。
- 按钮的普通、悬停、按下、禁用状态。
- 页签的普通、选中状态。
- 下拉选择框。
- 主音量滑杆轨道和滑块。
- 标题装饰、分隔线、返回按钮和必要的小型菜单标记。

组件需允许后续增加语言、键位和音效分轨选项，而不要求重画菜单底板或现有按钮。

## 6. 局内主体美术冻结边界

### 6.1 立即推进

- 酒桶、炖锅、烤架、勺子。
- Day 1-3 使用的材料和成品图标。
- Ryan 等当前确定出场人物立绘。
- 容器投料、成品弹出、右键退料的轻量 VFX。
- 标题页、设置面板和 Tavern 局内菜单共用的组件化 UI 美术。

### 6.2 程序侧冻结

- 每个容器的显示尺寸、投料碰撞口、抓取区和 `OutputAnchor`。
- 主体资产使用透明 PNG，作为独立节点接入。
- 工具和容器保持独立节点，不合并到酒馆背景。
- 后续新增功能优先补状态贴图、动画帧和 VFX，不推翻主体外形。

### 6.3 暂缓

- 完整酒馆背景最终稿。
- 把桌面工具固定位置画死的背景细节。
- 依赖未来玩法的工具架、复杂装饰和非菜单类局内 HUD 完整皮肤。

## 7. 数据流

### 7.1 右键退料

```text
玩家右键
  -> BarWorkspace 命中酒桶或炖锅
  -> 容器 pop_last_ingredient()
  -> 容器移除最后一份材料并重置加工进度
  -> BarWorkspace 在 OutputAnchor 生成 DeskItem
  -> DeskItem 向上弹出并沿用桌面物理、回收规则
```

### 7.2 设置

```text
启动
  -> GameManager 初始化 SettingsManager
  -> SettingsManager 读取 user://settings.cfg
  -> 应用窗口模式、窗口尺寸、Master 音量

玩家调整设置
  -> SettingsPanel 调用 GameManager.settings
  -> SettingsManager 更新、即时应用并保存
  -> 标题页和局内菜单下次打开时读取同一状态
```

## 8. 错误处理

- 设置文件不存在时使用默认值并正常启动。
- 设置文件字段缺失或值非法时回退到对应默认值。
- 分辨率仅接受白名单值。
- 主音量限制在 `0-100%`。
- Headless 环境不调用实际窗口变更。
- 容器退料接口在空容器时返回空字符串，不生成实体。

## 9. 验证

### 9.1 自动验证

- 酒桶右键连续退料遵循后进先出，并重置摇晃进度。
- 炖锅右键连续退料遵循后进先出，并重置搅拌进度。
- 空容器右键不生成物品。
- Tavern 场景不再包含 `WashBasin`。
- 教程不再提到清洗盆，改为右键退料。
- 设置默认值、非法值回退、保存和重新加载行为正确。
- 标题页存在“设置”按钮，不再依赖 `RestartButton`。
- 局内菜单可打开设置面板。
- 标题页、设置面板和 Tavern 局内菜单使用同一套可复用菜单组件。

### 9.2 标准编辑器手动验证

按项目主流程运行：

```text
TitleScreen -> DayMap -> Tavern -> LedgerScreen -> DayMap
```

额外检查：

1. 标题页打开设置，切换窗口化、全屏和三档分辨率。
2. 调整主音量，退出并重启游戏，确认设置保留。
3. 局内菜单打开设置并返回，确认回到局内菜单。
4. 向酒桶投入两种材料，连续右键两次，确认按倒序弹出。
5. 向炖锅投入两种材料并搅拌，右键一次，确认最后材料弹出且搅拌进度重置。
6. 确认清洗盆已从 Tavern 布局移除。
7. 全流程保持 `0 errors/warnings`。
