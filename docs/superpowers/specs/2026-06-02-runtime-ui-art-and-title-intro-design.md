# 运行时 UI 美术接入与标题页入场动效设计

> 日期：2026-06-02
>
> 范围：补齐设置控件、局内菜单、快捷栏、双栏背包和剧情对话的运行时美术，并为标题页增加分段入场动效。

## 1. 背景

标题页、毛笔菜单组件和设置面板已经有一套实际落地的视觉语言：深色毛笔底、浅色像素文字、琥珀色强调和短下划线标记。当前缺口主要是接入不完整：

- 设置面板里的分辨率等 `OptionButton` 仍使用 Godot 默认展开列表。
- 音量轨道已有毛笔素材，但滑杆仍混有默认控件外观。
- Tavern 快捷栏和双栏背包仍以基础占位样式为主。
- Tavern 局内菜单页签已经使用毛笔组件，但配方、背包和教程内容区仍需统一整理。
- 剧情对话仍调用 Dialogue Manager 自带示例气泡。
- 标题页已有背景、Logo、右侧四个菜单按钮和 Hover 标记，但没有完整入场编排。

本轮视觉依据是仓库内已经实现并确认的运行时资产，不再引用旧版 `docs/07_美术需求文档.md` 作为约束来源。

## 2. 目标

- 使用现有毛笔组件补齐运行时 UI，不重做已确认的标题页和菜单风格。
- 为设置面板的展开列表和音量滑杆接入完整美术。
- 为 Tavern 快捷栏和双栏背包接入美术，不改变库存和拖拽行为。
- 统一 Tavern 局内菜单的配方、背包和教程页面视觉。
- 将 Dialogue Manager 示例气泡替换为项目自有的毛笔风气泡。
- 为标题页增加“黑屏 -> 背景 -> Logo -> 右侧四按钮依次出现”的分段入场动效。

## 3. 非目标

- 不修改旧版美术需求文档。
- 不重做标题页背景、Logo、右侧菜单条、Hover 短标记或已有暖光效果。
- 不改变快捷栏 10 槽结构、点击取材行为、库存扣减或桌面生成流程。
- 不改变背包双栏结构、拖拽取出行为或剧情物品规则。
- 不改写 `.dialogue` 内容，不新增头像框，不制作视觉小说式大头像布局。
- 不直接修改 `addons/dialogue_manager/`。
- 不在本轮重做文档阅读页、顶栏、耐心条或其他未明确纳入的 HUD。

## 4. 视觉基准

统一复用当前已经落地的视觉语言：

- 深色毛笔纹理作为面板、条目和按钮底。
- 浅色像素文字作为正文。
- 琥珀色用于 Hover、当前选项、滑块和短标记。
- 选中态使用短下划线或短划线，不增加整圈琥珀边框。
- 素材加载失败时回退到深色代码样式，保证界面仍可操作。

已有 `menu_brush_panel.png`、`menu_brush_band.png`、`menu_brush_tab.png`、`menu_brush_slider.png`、`menu_brush_divider.png` 和标题页短标记优先复用。只有现有切片无法覆盖运行时需求时，才新增少量同风格切片。

## 5. 组件与场景边界

### 5.1 通用样式入口

扩展 `scripts/ui/theme_colors.gd`，集中提供以下样式能力：

- `OptionButton` 本体和其 `PopupMenu` 展开列表。
- `HSlider` 的轨道、滑块和禁用态。
- 快捷栏背景、快捷栏槽位和槽位 Hover。
- 背包主面板、双栏标题和可拖动条目。
- 对话面板、角色名、正文和选项按钮。

场景脚本调用样式入口，不在各个 UI 脚本里重复拼接主题配置。

### 5.2 设置面板

保留 `scenes/ui/SettingsPanel.tscn` 的现有结构和 `SettingsManager` 数据流。

- `Mode` 和 `Resolution` 展开时，`PopupMenu` 使用深色背景、像素字体、浅色正文和琥珀 Hover/当前项。
- `VolumeTrack` 继续使用已有毛笔轨道。
- `Volume` 隐藏 Godot 默认轨道，使用同风格琥珀滑块。
- 拖动滑杆时继续即时更新百分比并调用 `SettingsManager.set_master_volume_percent()`。

### 5.3 Tavern 局内菜单

保留现有 `OverlayMenu` 结构和页签行为：

- 页签：配方、背包、设置、教程。
- 操作按钮：整理桌面、关闭。
- 配方内容区和教程入口使用统一像素字体、间距和毛笔风条目视觉。
- 背包页签继续打开独立可拖拽背包弹窗，不恢复菜单内只读背包作为主要入口。

### 5.4 快捷栏

保留 Tavern 底部 `ShortcutBar` 的 10 槽结构和 `BarWorkspace` 点击取材链路。

- 背景使用现有快捷栏底图或同风格切片。
- 每个槽位接入统一槽位贴图。
- 有物品时显示图标、名称和当前数量。
- 图标缺失时保留当前颜色块占位。
- Hover 时增加轻微琥珀强调，不增加复杂动画。

### 5.5 双栏背包

保留 `InventoryOverlay` 的双栏结构：

- 左栏：材料。
- 右栏：剧情物品。
- 条目继续由 `InventoryDragRow` 承担拖拽入口。
- 面板、栏标题和条目使用毛笔风样式。
- 条目显示图标、名称和数量。
- Hover 时使用文字提亮或短标记。
- 从面板外放下物品时，继续走现有 `item_dropped` 信号和桌面生成入口。

### 5.6 剧情对话

复制 Dialogue Manager 示例气泡为项目自有场景和脚本，放入 `scenes/ui/` 与 `scripts/ui/`。不修改插件目录。

项目气泡保留插件示例气泡已有的：

- 逐字显示。
- 点击推进。
- 快进。
- 选项显示和选择。
- 输入阻塞配置。

只替换视觉：

- 深色毛笔面板。
- 像素字体角色名和正文。
- 毛笔条选项按钮。
- 琥珀 Hover 和当前选项强调。
- 气泡短淡入。

场景里已经有客人立绘，因此本轮不增加头像框。

`GameManager` 不再调用 `show_example_dialogue_balloon()`，改为实例化项目气泡并调用其 `start()`。对话资源、起始标题和 `extra_states` 保持不变。

### 5.7 标题页

保留标题页现有资产和微动效，只增加一次性入场编排。

初始状态：

- 全屏黑色遮罩可见。
- 背景、暖光层、Logo、右侧菜单条和四个按钮透明。
- 四个按钮临时禁用。

播放顺序：

1. 黑屏停留 `0.4s`。
2. 背景和暖光层在 `0.7s` 内淡入。
3. Logo 在 `0.5s` 内淡入，并从略低位置上浮到现有位置。
4. 右侧菜单条显示。
5. 四个按钮从上到下依次淡入，每项间隔 `0.1s`。
6. 入场结束后解锁按钮，恢复现有 Logo 轻微浮动、暖光呼吸和 Hover 短标记。

入场 Tween 在离开场景时必须可清理。动画结束前不响应按钮点击。

## 6. 数据流

### 6.1 设置

```text
SettingsPanel
  -> SettingsManager
  -> 立即应用窗口模式、窗口尺寸和 Master 音量
  -> 保存 user://settings.cfg
```

设置 UI 不维护独立真相源。

### 6.2 快捷栏与背包

```text
GameManager.inventory
  -> ShortcutBar / InventoryOverlay 显示
  -> 点击快捷槽或拖出背包条目
  -> BarWorkspace.spawn_inventory_item_at()
  -> 现有库存扣减和桌面实体生成逻辑
```

本轮只增加视觉节点和显示信息，不新增库存写入口。

### 6.3 对话

```text
GameManager 加载 *.dialogue
  -> 实例化项目自有 DialogueBalloon
  -> start(dialogue_resource, "start", extra_states)
  -> Dialogue Manager 继续驱动文本、选项和结束信号
```

## 7. 错误回退

- 毛笔纹理缺失：回退到 `ThemeColors` 提供的深色 `StyleBoxFlat`。
- 物品图标缺失：显示当前颜色块占位，不隐藏条目。
- PopupMenu 样式应用失败：保留可用的默认弹出列表，不阻塞设置。
- 项目气泡实例化或启动失败：沿用 `GameManager` 现有对话失败恢复逻辑，解除对话模式，避免剧情卡死。
- 标题入场 Tween 被切场景打断：停止 Tween 并清理引用，不写入存档状态。
