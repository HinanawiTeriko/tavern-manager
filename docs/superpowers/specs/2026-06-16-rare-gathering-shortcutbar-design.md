# 稀有采集材料与快捷栏绑定设计

> 日期：2026-06-16
> 范围：DayMap 采集价值重构、第一批稀有材料/升级菜、背包拖入快捷栏、后续物品图标资产管线。

## 1. 背景

当前基础材料开局库存很高，商店材料价格低，而采集一次通常只获得 1 个普通材料并消耗 1-2 点行动力。行动力还要与告示板、矿道、米拉货摊、掮客等剧情地点竞争，所以理性玩家几乎不会把行动力花在普通采集上。

这不是单纯数值问题，而是采集缺少独特职责。新设计把普通经营材料交给稳定渠道，把采集改成“稀有机会来源”：玩家不采集也能正常开店，但采集能拿到更高利润、额外对白、委托菜和未来 NPC 的替代解法。

## 2. 设计目标

- 让采集点提供普通材料之外的明确诱因。
- 不破坏 Ryan/Mira 已经自洽的主线因果。
- 不改现有容器上限和三容器手感。
- 稀有材料优先做现有菜肴的升级版，降低学习成本。
- 玩家采到稀有材料后，可以自然地把它放入快捷栏使用。
- 物品美术走项目像素资产管线，不把 AI 原图直接接进运行时。

## 3. 非目标

- 不把稀有材料做成 Ryan/Mira 结局的硬钥匙。
- 不把烤架改成双材料容器。
- 不把所有新增材料都变成日常库存压力。
- 不重做 DayMap UI 或 Tavern 快捷栏节点结构。
- 不在第一版实现天气、工具升级、复杂掉落表或多稀有池轮换。

## 4. 推荐方案

普通材料保持稳定供应：初始库存、商店、采集稳定产出都可以存在。采集点新增一个主稀有材料，稀有材料用于现有菜肴的升级版、当天委托菜和少量剧情回声。

稀有材料对剧情只做软影响：

- 可以让 NPC 多一句、更早露出情绪或给少量好感。
- 可以触发小委托、传闻或额外账本记录。
- 不能替代关键线索、信任判断、递交道具和兜底行动。

## 5. 第一批材料与菜谱

第一版优先接入 4 个现有 repeatable 采集点，避免先新增地图节点。第五类矿盐/冷泉等材料留到第二批或新地点时再接入。

| 地点 | 稳定产出 | 主稀有材料 | 升级菜 | 容器规则 |
| --- | --- | --- | --- | --- |
| 菌菇林地 `mushroom_forest` | 草药 | 洞窟菌 `cave_mushroom` | 菌菇肉汤 `cave_mushroom_stew` | 生肉 + 洞窟菌，炖锅 |
| 暗河沿岸 `dark_river` | 生肉 | 岩蜥肉 `rock_lizard_meat` | 岩蜥烤排 `rock_lizard_steak` | 岩蜥肉，烤架单材料 |
| 葡萄藤架 `grape_trellis` | 葡萄 | 北路酸葡萄 `north_sour_grape` | 旧路酸葡萄酒 `old_road_wine` | 北路酸葡萄，酒桶单材料 |
| 农庄磨坊 `mill_farm` | 麦芽 | 黑麦芽 `black_malt` | 矿工黑啤 `miner_dark_ale` | 黑麦芽，酒桶单材料 |

第二批候选：

| 来源 | 稀有材料 | 升级菜 | 备注 |
| --- | --- | --- | --- |
| 新冷泉点或暗河扩展 | 冷泉水 `cold_spring_water` | 冷泉草药茶 | 需要决定是否新增地点，第一版不混入暗河双稀有池 |
| 矿道边缘资源点 | 矿盐 `mine_salt` | 咸肉汤或未来调味 | 不复用 `abandoned_mine` 的一次性剧情调查合同 |
| 蜂巢/市集特别货 | 萤火蜂蜜 `glow_honey` | 蜜麦面包 | 适合未来日常委托菜 |

## 6. 掉落规则

每个采集点访问后：

- 必得稳定普通材料，第一版建议 `x2`。
- 有 35% 概率额外获得主稀有材料 `x1`。
- 同一地点连续 2 次未出稀有后，第 3 次必出。
- 保底计数按地点保存，跨天和存档保留。
- Day2 菌菇林地的 `sleep_powder` 剧情奖励保持特殊规则，不进入普通稀有池，也不消耗洞窟菌保底。

奖励结果需要返回聚合字典给现有 gathering toast，让玩家看到“采集获得：草药 x2、洞窟菌 x1”。

## 7. 地点描述改写

地点文案需要告诉玩家普通产出和可能的稀有机会，但保持自然口吻。

| 地点 | 新描述方向 |
| --- | --- |
| 菌菇林地 | 潮湿洞穴里有草药，也偶尔能在腐木背面找到肥厚的洞窟菌。 |
| 暗河沿岸 | 暗河旁能捕到肉食，也有岩蜥在温热石缝里出没。 |
| 葡萄藤架 | 普通葡萄容易采，深处偶尔结着酸涩的北路野葡萄。 |
| 农庄磨坊 | 磨坊能换到麦芽，旧仓里偶尔翻出颜色更深的黑麦芽。 |

详细面板还应显示“可能发现：洞窟菌”等短提示，但不需要新增复杂 UI。

## 8. 快捷栏绑定

现状：`BarWorkspace._init_material_slots()` 从库存材料自动排序填充 `ShortcutBar/Slot0..9`。稀有材料变多后，自动排序会让玩家难以控制今晚要用什么。

新规则：

- `ShortcutBar/Slot0..9` 节点路径保留。
- 快捷栏槽位保存的是 item key 绑定，不保存物品实例、不复制库存。
- 库存数量仍来自 `InventorySystem`。
- 背包格子拖到快捷栏槽位时，绑定该 item key。
- 如果同一个 item key 已绑定在别的槽位，旧槽位清空，避免重复。
- 如果目标槽位已有其他 item key，则直接替换。
- 绑定物品库存为 0 时，槽位可保留但显示为不可用，点击不生成物品。
- 只允许材料和调料类物品绑定；剧情物品和成品不能绑定到快捷栏。
- 初始绑定由现有基础材料顺序生成：麦芽、葡萄、面粉、生肉、草药，其余槽为空。

交互入口优先复用现有背包拖拽路线：

1. `InventoryGridSlot` 继续提供 `{ "item_key": key }` drag payload。
2. `InventoryOverlay.item_dropped(item_key, global_position)` 继续作为拖出背包的公共信号。
3. `TavernView` 或 `BarWorkspace` 先判断 `global_position` 是否落在快捷栏槽位。
4. 命中快捷栏则写绑定并刷新快捷栏；未命中则保持现有拖到世界生成物理物品行为。

## 9. 数据与保存

需要新增或扩展的数据：

- `data/items.json`：新增稀有材料和升级成品。
- `data/recipes.json`：新增升级菜谱，不超过现有容器材料上限。
- `data/locations.json`：采集点新增主稀有配置、地点描述和稳定产出数量。
- GameManager 或 DayMapSystem 状态：保存地点保底计数。
- Save/Load：保存快捷栏绑定和稀有保底计数。

不建议把保底计数做成纯临时内存状态，因为玩家保存退出后会丢失采集承诺感。

## 10. 美术与资产管线

第一版玩法可先用临时色块跑通，但正式交付必须补齐图标资产。

运行时路径沿用现有 icon 读取习惯：

- 稀有材料：`assets/textures/tavern/icons/{item_key}.png`
- 升级成品：`assets/textures/tavern/items/{product_key}.png`

资产管线：

- AI 原图放入 `art_sources/generated_raw/rare_gathering/`。
- 清理后的 native 像素源放入 `assets/source/rare_gathering/`。
- 运行时 PNG 由 deterministic exporter 输出到现有 runtime 路径。
- 新增 `scripts/tools/export_rare_gathering_assets.py`。
- 新增 `scripts/test/test_rare_gathering_asset_pipeline.py`，验证 runtime PNG 是 native PNG 的整数最近邻导出。
- 输出 contact sheet 到 `docs/art/rare_gathering_contact_sheet.png`。
- 图标不能包含可读文字、数字、logo 或 UI 状态。

## 11. 分阶段实现

### 阶段 A：规则与测试

- 为 DayMap 采集奖励写测试：稳定产出 x2、35% 稀有、保底、Day2 `sleep_powder` 特例不破坏。
- 为 recipes/items 写测试：新增 key 可加载，容器上限不被突破，烤架升级菜是单材料。
- 为保存写测试：保底计数和快捷栏绑定可 roundtrip。

### 阶段 B：数据与采集

- 新增 4 个稀有材料和 4 个升级成品。
- 改写 4 个采集点描述。
- 实现稀有掉落和保底。
- 更新采集 toast 聚合显示。

### 阶段 C：快捷栏绑定

- 新增快捷栏绑定状态。
- 改 `BarWorkspace._init_material_slots()` 从绑定读取槽位，而不是每次按库存自动排序。
- 复用背包拖出信号，将落点命中快捷栏时转成绑定。
- 保留 `ShortcutBar/Slot0..9` 节点和现有点击生成物理物品行为。

### 阶段 D：资产

- 生成并整理第一批 8 个图标：4 个稀有材料、4 个升级成品。
- 通过 exporter 输出 runtime PNG。
- 跑 pipeline 测试并生成 contact sheet。

### 阶段 E：剧情软反馈

- 只添加少量对白或账本回声，不改 Ryan/Mira 主线判定。
- 米拉可对 `old_road_wine` 多一句旧路味道反馈。
- 未来 NPC 可以把升级菜作为好感捷径或替代解法。

## 12. 验证范围

必须跑：

- `test_day_map_system.tscn`
- `test_day12_smoke.tscn`
- `test_workspace_scene_recovery.tscn`
- `test_inventory_system.tscn`
- 新增稀有采集测试场景
- 新增资产 pipeline 测试

需要人工手测：

- 白天采集 3 次同一地点触发保底。
- 采到稀有材料后打开背包，拖到快捷栏。
- 快捷栏绑定后点击生成物理物品。
- 库存归零时绑定槽位不可用但不崩。
- 用升级菜服务普通订单不会破坏现有订单/结算。

## 13. 风险与边界

- 快捷栏是 Tavern 关键 UI 合同，不能改节点路径或槽位数量。
- 保底计数如果不入存档，会造成玩家感知损失。
- 稀有材料太多会增加认知负担，第一版只落 4 个。
- 美术资产量会快速膨胀，必须先确认数据 key 和菜谱再批量生成。
- 稀有菜如果给太多好感，会间接破坏剧情平衡，第一版只做对白/利润轻反馈。

## 14. 自查

- 没有未决事项或空白章节。
- 第一版范围集中在 4 个现有采集点，没有同时新增地图大区域。
- 容器规则与现有实现一致：酒桶/炖锅可双材料，烤架单材料。
- 快捷栏绑定不复制库存，库存仍由 `InventorySystem` 单一来源管理。
- Ryan/Mira 主线因果不依赖稀有材料。
