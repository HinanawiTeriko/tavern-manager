# 物理调查场景：废弃矿道试点（概念切片）

> 日期：2026-06-02
> 状态：概念已确认，待技术 spike（实现延后，本 spec 为设计存档）
> 目标：验证「把涉及剧情的调查放进独立界面、复用重力系统让玩家亲手收集」能否真正带来调查感；并以废弃矿道为单场景试点，量出一个英雄场景的真实成本。
> 上游：承接 [[2026-06-02-daytime-lead-gating-ryan-opening-design]]——该 spec 让调查地点按线索逐格出现；本 spec 进一步把「访问→文字弹窗」的调查结果，换成一个物理场景。

## 1. 动机

当前调查 = 点「访问」→ 弹窗读结果。那是**读一份调查报告**，不是调查。调查感的本质是**亲手发现的能动性**——掀开、翻找、自己挖出线索。文字弹窗永远给不了，因为信息是被递给你的，不是被你挖出来的。把「发现」从一行文字变成一个物理动作，调查才有体感。

## 2. 与夜晚的关系 / 北极星

- **复用引擎、不撞玩法**：夜晚物理动词是「创作/表达」（扔料、摇桶、调酒）；调查物理动词是「搜寻/揭示」（翻找、扒开、捡起）。同一个重力引擎，两个不同的情感寄存器，不冗余、不抢戏。
- **守北极星（白天喂利害、不预定夜晚）**：调查场景收的是**线索/知识**，**不是材料**。一旦在里面物理捡材料，就滑回「风险探索」、跟夜晚抢戏。分工：采集留在轻列表（沙盘底噪），调查进物理场景（故事主菜）。
- 这也回应了最初的「采集无聊」：不再硬给采集加戏，把戏放到本该有戏的调查里。

## 3. 关键设计决策：复用「积木」，不复用「房间」

权衡过两条路：

- **少而精的英雄场景（定制、稀有）**：体验上限最高、节奏天然好（稀有=隆重）、最配叙事驱动的游戏；但单个贵、不可扩展。
- **数据驱动的可复用搜索模板（通用、普及）**：可扩展、单个便宜；但同质化、节奏疲劳（每条线索都翻箱子→无聊），且通用模板会把特定剧情的质感磨平——而那是剧本作者的核心价值。

**决策：走少而精的英雄场景，但用一套共享「积木」压低单个成本。**

> 复用「积木」（可拾取/可翻开物件 + 线索藏匿 + 搜索交互组件），英雄场景在积木上手工摆布局和氛围。
> **积木让定制变便宜；房间模板让定制消失。** 这与夜晚架构同构——夜晚不是一个写死的吧台，是一套物理积木反复搭出不同的酒。

物理调查只留给**关键剧情节拍**，多数小线索仍走轻列表。

## 4. 积木词汇（这一个场景要验证的候选集）

| 积木 | 作用 | 复用现有? |
|---|---|---|
| 可抓取 + 重力 | 捡起、拖动物件 | ✅ 夜晚拖拽+重力 |
| 可扒开的遮蔽物 | 拖开碎石/塌方，露出底下的东西 | 🟡 简单新增（核心调查动词） |
| 容器倾倒/抖落 | 抖背包，内容物哗啦撒出 | ✅ 酒桶摇晃机制 |
| 检视→一行观察 | 捡起某物给一句环境叙事 | 🟡 轻量新增 |
| 线索物→开文档 | 捡起线索 → 打开 DocumentOverlay | ✅ 桥到现有 DocumentSystem |

只有 2 块是新的，且都很轻；其余全是夜晚物理 + 文档系统的复用。

## 5. 废弃矿道场景设计

### 5.1 脊柱（一句话）

> 上一支接了血斧「护送」委托的人，死在这里。玩家亲手扒开掩埋的东西，挖出那份沾血委托书，意识到莱恩明天要接的是同一个绞肉局。

物理动作与情感咬合：**扒开碎石 = 把被掩埋的真相挖出来**；**抖出背包 = 真相哗啦掉出来**。

### 5.2 浅层 / 深层（「可错过」的载体）

不强制找到线索，于是场景分两层，让「翻得多深」成为差异：

- **浅层（显眼、随手）**：断箭、凹陷的盾、掉落的靴。捡起=一句观察（「被打崩了」）。匆忙的玩家拿到浅层读数就走。
- **深层（要刻意挖）**：血迹尽头的塌方碎石 → 扒开 → 撕裂的背包 → 抖出沾血委托书。深挖的玩家才挖出铁证。

错过不是 gotcha，是「你没多翻那一下」——挣来的代价，把「知道 vs 准备」的尽职轴物理化。

### 5.3 可错过的连锁后果（系统已支持）

现有数据中公会柜台 `requiresRead: bloodied_contract`。所以：

> 矿道没挖到委托书 → 没读过它 → 公会柜台不开 → 拿不到「替代委托」→ 当晚既缺情报、又少一个给莱恩保命的选项。

白天翻得多深，直接决定今晚有几张牌可打。非新机制，是现有 `requiresRead` 链条自然兑现。

### 5.4 防「闷亏」纪律

离开时给**软提示**（血迹仍延向未翻处 / 一句「总觉得还藏着什么」），让「错过」是选择不是 bug；不强制、只耳语。

### 5.5 空间布局（侧视横截面，复用重力，单屏不滚动）

```
 ┌──────────────────────────────────────────────────────────────┐
 │ ░ 顶光                                       ▓▓▓▓▓▓▓▓▓▓▓▓▓▓   │
 │ ░                                            ▓▓  塌方碎石  ▓▓   │  ← 深层
 │                                              ▓▓▓▓▓▓▓▓▓▓▓▓▓▓   │
 │     [断箭]   [凹盾]   [破靴]                  ▓▓ (撕裂背包) ▓   │
 │   /                                          ▓▓▓▓▓▓▓▓▓▓▓     │
 │  / 入口   · · 血迹 · · · · · · · · · · · ▶   ▓▓▓▓▓▓▓        │
 ├──────────────────────────────────────────────────────────────┤
 │  地面 ▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁▁  [离开 ▸]   │
 │        浅层散落物         血迹引导动线        碎石+埋藏背包      │
 └──────────────────────────────────────────────────────────────┘
```

左半浅层（随手），右半深层（要挖），血迹是连接两者、把视线从入口拽到碎石的动线。

### 5.6 玩家动线

1. 淡入，看见血迹。
2. 顺手捡浅层散落物（得到「打崩了」的读数）。
3. 血迹尽头扒开碎石 → 露出背包。
4. 抓背包、倾倒，内容物（硬币·血斧队牌·沾血纸）哗啦落地（小重力演出）。
5. 捡起沾血纸 → DocumentOverlay 全开 = 沾血委托书 → 重锤。
6. （若径直点「离开」未挖碎石）→ 软提示 → 带着残缺过夜。

整段 30–60 秒，一个核心动作，体力照旧花 1。

## 6. 范围

### In scope（本 spec）
- 概念验证 + 废弃矿道单场景的体验/积木/布局设计。

### Out of scope（明确延后）
- **现在就建**：本 spec 仅存档，不碰代码。
- **定 pattern**：先做单场景 spike，别一上来就把它推广成模式。
- **美术**：英雄场景的独立美术是真正的成本上限，待 spike 量化。
- 其余调查点（告示板/公会）是否也物理化——待矿道试点结论。
- 环境观察的具体台词（断箭/队牌/委托书正文）——剧本作者后续填，本 spec 占位。

## 7. 下一步

1. **技术 spike：只做废弃矿道一个场景**，目的只为量真实成本——2 块新积木好不好做、单场景美术多贵、手感对不对。
2. spike 跑通后再决定：是否把「英雄场景」扩成可复用积木上的一种模式、覆盖哪些节拍。
3. 按项目规矩，建场景是独立功能，单开对话/分支走 spec→plan→code。
