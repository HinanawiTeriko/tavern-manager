## 六、数据文件

### `data/items.json`

定义物品名、颜色、价格、类型和 profile 引用。当前关键类型：

| 类型 | 说明 |
|------|------|
| `material` | 可从快捷栏取出的基础材料 |
| `intermediate` | 中间/焦糊/特殊加工结果，通常不进入订单池 |
| `product` | 可上菜成品 |
| `special` | 剧情或特殊物品 |

当前基础材料：`flour`、`meat_raw`、`ale`、`grape`、`herb`。

当前可上菜成品：`bread`、`meat_cooked`、`ale_beer`、`wine`、`herb_tea`、`meat_sand`、`herbal_ale`、`spiced_wine`、`meat_stew`、`herb_broth`、`malt_porridge`。

### `data/recipes.json`

当前结构是 product key 到配方数据：

```json
{
  "wine": {
    "name": "葡萄酒",
    "ingredients": ["grape"],
    "container": "barrel",
    "memory_for": { "mira": "day4_road_story" }
  }
}
```

`CraftSystem` 以 `container + sorted(ingredients)` 建索引。

### `data/item_physics_profiles.json`

包含三个 profile 分区：

| 分区 | 用途 |
|------|------|
| `physics` | mass、friction、bounce、damp、gravity |
| `collision` | rect/circle/capsule 形状参数 |
| `feedback` | impact 声音/粒子/抖动预留 |

### 其它 gameplay 数据

| 文件 | 说明 |
|------|------|
| `barrel.json` | 酒桶摇晃阈值和品质相关参数 |
| `craft_style_thresholds.json` | L3 风格阈值 |
| `locations.json` | DayMap 地点 |
| `npcs.json` | NPC 出现条件、对话和风格偏好 |
| `inventory_default.json` | 初始库存 |
| `shop.json` | 商店配置 |
| `seasonings.json` | 旧香料数据 |
| `operations.json` / `combines.json` | 旧合成链数据，当前三容器主流程主要读 `recipes.json` |

## 九、输入与操作

| 操作 | 方式 |
|------|------|
| 打开菜单 | `E`，映射为 `menu_toggle` |
| 取材料 | 左键点击快捷栏槽位 |
| 拖拽物理体 | 左键按住物体/容器/勺子 |
| 酒桶加工 | 投料后摇晃酒桶 |
| 烤架加工 | 把材料按在烤区，离开后定稿 |
| 炖锅加工 | 投料后用勺尖在锅内移动 |
| 上菜 | 成品拖进 `CustomerDropArea` 松手 |
