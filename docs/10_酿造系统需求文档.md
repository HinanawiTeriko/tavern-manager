## 1. 概述

本次涉及两个关联功能：

- **Part A**：香料系统改造——删除独立 SeasoningZone，香料撒入合并到主合成区，新增食物属性系统
- **Part B**：酿造系统——全新增殖，白天大地图上管理酿造桶，时间推进 + 随机事件 + 神秘配方

两者关联：酿造产物可撒香料调味，再卖给客人。

## 2. Part A — 香料系统改造

### 2.1 当前问题

- 独立的 `SeasoningZone` 将香料撒入与主合成流程割裂
- 香料只有口味标签，无游戏性效果
- 成品拖来拖去体验繁琐

### 2.2 改动方案

#### 删除 SeasoningZone

- 移除 `scripts/ui/seasoning_zone.gd` 及相关场景节点
- 香料操作统一在主合成区（MixingArea / CraftStation）完成

#### 新交互：移动撒入

**流程**：

1. 玩家从快捷栏选中一种香料
2. 在地图/合成区上方移动光标（香料的虚拟罐子跟随）
3. 合成区下方出现**香料进度条**
4. 移动过程中进度条持续填充
5. 进度条满 → 消耗 1 份香料 → 给成品附加对应的食物属性

**规则**：
- 合成区有东西时才能撒，无东西时进度条不涨
- 进度条约 2-3 秒移动即可填满
- 中途松开/移出区域 → 进度条重置，不消耗香料
- **覆盖式叠加**：同一成品只能有一个属性，后撒的覆盖前者
- 属性与香料的对应关系见下

#### 食物属性系统

| 香料 Key | 名称 | 属性标签 | 属性效果 |
|-----------|------|----------|----------|
| `spice` | 辣 | 辛辣 | 口味标记（暂无特殊效果） |
| `herb_spice` | 香草 | 清香 | 口味标记（暂无特殊效果） |
| `salt` | 盐 | 咸香 | 口味标记（暂无特殊效果） |
| `sleep_powder` | 迷睡花粉 | 安眠 | 口味标记 + 叙事触发条件（Ryan 剧情线） |

属性存储在成品数据中，随物品传递。客人结算时可用属性触发特殊对话/叙事分支。

### 2.3 进度条视觉

- 位置：合成区正下方，宽约 200px，高 12px
- 进度方向：左→右 填充
- 颜色：香料对应颜色渐变（辣=红、香草=绿、盐=白、花粉=紫）
- 满格时闪烁一次作为反馈

### 2.4 影响范围

| 文件 | 操作 | 说明 |
|------|------|------|
| `seasoning_zone.gd` | 删除 | 不再需要 |
| `Tavern.tscn` | 修改 | 移除 SeasoningZone 节点 |
| `craft_station.gd` | 修改 | 新增香料撒入逻辑、进度条 |
| `seasoning_system.gd` | 扩展 | 新增食物属性查询方法 |
| `game_manager.gd` | 修改 | 移除 SeasoningZone 相关引用 |
| `seasonings.json` | 扩展 | 新增 `attribute` 字段 |
| `items.json` | 可能扩展 | 成品数据结构加 `seasoning` 字段 |

## 3. Part B — 酿造系统

### 3.1 系统定位

- 新子系统 `BrewSystem`（RefCounted），由 GameManager 持有
- 酿造桶放在**白天大地图**（DayMap），酒馆建筑后方
- 时间酿造：投入材料 → 等待天数 → 取出成品
- 配合随机事件 + 神秘配方增加策略深度

### 3.2 核心循环

```
白天大地图 → 投入材料到酿造桶 → 推进天数 → 桶进度推进 → 随机事件 → 取出成品 → 夜晚卖给客人
```

### 3.3 酿造桶

#### 数量

- 初始 2 个
- 可通过商店/剧情解锁至最多 4 个

#### 界面

```
┌─────────────────────────────────┐
│  酿造桶 #1           [X 关闭]    │
│─────────────────────────────────│
│                                 │
│   材料槽 [空]  [空]  [空]        │  ← 最多 3 种材料，每种 1-5 个
│                                 │
│   投料比例: 麦芽 2 : 葡萄 1      │  ← 实时显示
│                                 │
│   产物名称: 麦芽酒               │  ← 已知配方显示名，未知显示"???"
│                                 │
│   预计天数: 1 天                 │
│   已过天数: ████░░░░░░ 0/1      │  ← 进度条
│                                 │
│   当前状态: 正常发酵中            │
│                                 │
│          [开始酿造]              │
│          [取出(未完成)]          │
└─────────────────────────────────┘
```

#### 酿造规则

- 每个桶最多投 **3 种材料**，每种可投 **1-5 个**
- 材料配比决定产物（见配方表）
- 已知配方自动匹配显示产物名，未知配方显示 `"???"`
- 投入后点击"开始酿造"，桶进入酿造状态
- 酿造中不可投料，只能等完成或手动取出（材料损失）

#### 品质与时机

| 取出时机 | 品质 | 售价 | 说明 |
|----------|------|------|------|
| 到达 1 天（设定天数） | 普通 | 标准 | 基础 |
| 到达 2 天 | 陈酿 | ×1.5 | 等一天 |
| 到达 3 天 | 极品 | ×2 | 等两天 |
| 超过 4 天 | 变质 | 报废 | 材料全损 |

品质在取出时确定，显示在成品上。

#### 体力消耗

- 投入材料：**不消耗体力**
- 开始酿造：**不消耗体力**
- 取出成品：**不消耗体力**
- 处理随机事件：**消耗 1 点体力**（见 3.4 节）
- 手动提前取出（放弃酿造）：**不消耗体力**

机制意图：鼓励玩家每天去检查桶状态，但处理的体力成本让资源分配有取舍。

### 3.4 随机事件

酿造期间每天约 20-30% 概率触发事件。在大地图上桶有状态标记（冒泡/裂缝/发光等），玩家点击桶面板触发。

| 事件 | 概率 | 效果 | 处理方式 | 体力消耗 |
|------|------|------|----------|----------|
| **桶漏了** | 25% | 进度暂停，不处理会损失材料 | 消耗 1 个任意采集材料修补 | 1 |
| **异常发酵** | 20% | 品质降一级（极品→陈酿→普通） | 搅拌一下，瞬间处理 | 1 |
| **虫子入侵** | 15% | 材料直接报废 | 无可挽回，只能清空 | 1 |
| **稀有酵母** | 10% | 品质升一级，省一天 | 无需处理，自动生效 | 0 |
| **疯狂发酵** | 10% | 当前天数直接 +1（提前成熟） | 无需处理，自动生效 | 0 |
| **无事件** | 20% | — | — | — |

事件在玩家**打开桶面板时**触发和展示（不是大地图上自动弹窗）。

### 3.5 配方

#### 已知配方（初始解锁）

| 配方 | 材料配比 | 天数 | 售价 | 类型 |
|------|----------|------|------|------|
| 麦芽酒 | 麦芽 | 1 | 5 金 | 单材料 |
| 葡萄酒 | 葡萄 | 1 | 5 金 | 单材料 |
| 草药麦酒 | 麦芽 + 草药 (1:1) | 1 | 10 金 | 双材料 |
| 香料红酒 | 葡萄 + 草药 (1:1) | 1 | 11 金 | 双材料 |

#### 神秘配方（玩家发现）

| 配方 | 材料配比 | 天数 | 售价 | 说明 |
|------|----------|------|------|------|
| 幽灵酒 | 葡萄 + 草药 + 沉睡花粉 (1:1:1) | 2 | 18 金 | 暗紫色酒液，微微发光 |
| 烈焰麦酒 | 麦芽 + 香料 (3:1) | 2 | 15 金 | 红色的烈性酒，酒杯口有火焰特效 |
| 秘药汤 | 草药 + 生肉 + 面粉 (1:1:2) | 2 | 20 金 | 浓稠炖煮酿造物 |

**发现机制**：
- 第一次配出正确比例时触发"发现新配方！"提示
- 桶上闪过特殊光效
- 配方永久解锁，记录在菜单的配方册

### 3.6 与香料系统的联动

- 酿造产出的成品可以在夜晚 Tavern 酒吧合成区撒香料调味
- 撒上香料的酿造品同样带有食物属性标签
- 沉睡花粉酿造的"幽灵酒"自带"安眠"属性，不需要再撒花粉

### 3.7 DayMap 集成

```
DayMap 场景
│
├── ... (现有地图区域)
│
└── BrewingArea (新增)
    ├── 酒馆建筑后方区域（不可采集，纯操作区）
    ├── Barrel_1 (酿造桶 #1)
    │   ├── 桶的精灵图
    │   └── 状态标记（进度环/事件图标）
    ├── Barrel_2 (酿造桶 #2)
    └── ... (最多 4 个)
```

- 点击桶 → 弹出酿造面板（Control overlay）
- 每天的采集/商店操作完成后，可以来酿造区投放材料和取货
- 体力不足时仍可查看桶状态，但不能处理事件

## 5. 接入方式

### 新文件

| 文件 | 类型 | 说明 |
|------|------|------|
| `scripts/systems/brew_system.gd` | RefCounted | 酿造系统核心逻辑 |
| `scripts/ui/brew_panel.gd` | 脚本 | 酿造桶面板 UI |
| `scenes/ui/BrewPanel.tscn` | 场景 | 酿造面板 |
| `data/brewing.json` | 数据 | 酿造配置 |
| `assets/textures/ui/barrel.png` | 美术 | 桶精灵图 |
| `assets/textures/ui/barrel_event_*.png` | 美术 | 事件状态精灵图 |
| `assets/textures/ui/bar_spice_progress.png` | 美术 | 香料进度条 |

### 修改文件

| 文件 | 操作 | 说明 |
|------|------|------|
| `game_manager.gd` | 扩展 | 添加 BrewSystem 构造和持有 |
| `day_map_view.gd` | 扩展 | 添加酿造区 UI |
| `DayMap.tscn` | 扩展 | 添加酿造区节点 |
| `craft_station.gd` | 修改 | 新增香料撒入交互 |
| `seasoning_system.gd` | 扩展 | 新增食物属性查询 |
| `seasoning_zone.gd` | **删除** | 移除独立香料区 |
| `Tavern.tscn` | 修改 | 移除 SeasoningZone 节点 |
| `data/seasonings.json` | 扩展 | 新增 attribute 字段 |

### 信号

BrewSystem 暴露：

| 信号 | 参数 | 说明 |
|------|------|------|
| `barrel_state_changed` | `barrel_index, state` | 桶状态变化（空闲/酿造中/完成） |
| `event_triggered` | `barrel_index, event_type` | 随机事件触发 |
| `recipe_discovered` | `recipe_id` | 新配方发现 |

### 架构约束

- BrewSystem 不直接引用其他子系统，通过 GameManager 中转
- 成品产出后走 GameManager 库存入口
- 售价由 EconomySystem 统一处理，BrewSystem 只提供基础售价
