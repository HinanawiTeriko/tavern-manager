# 美术与资产管线

## 原则

- 运行时只引用处理后的游戏资源，不直接引用 AI 原图。
- 动态文本必须由 Godot Label/Button 渲染，不烘焙进图片。
- 新 UI/角色/背景资产要有源文件、导出脚本或 manifest、测试或 contact sheet。
- 不按 alpha、颜色或连通域猜切图；裁切必须来自 manifest 或明确矩形。
- 不在 prompt 中写现有游戏名或在世艺术家名。

## 目录约定

| 路径 | 用途 |
|---|---|
| `art_sources/generated_raw/` | 原始生成图、prompt 记录、未处理源 |
| `assets/source/` | 生产源图、native pixel 图、manifest |
| `assets/textures/` | Godot 运行时纹理 |
| `docs/art/` | contact sheet、报告 |
| `scripts/tools/` | 导出/归一化脚本 |
| `scripts/test/` | Python 管线测试 |

## 像素管线

全屏/大背景通常使用 native pixel 源，再整数 nearest-neighbor 放大到运行时分辨率。已有参考：

- Title：`scripts/tools/export_title_screen_assets.py`
- Intro：`scripts/tools/export_intro_assets.py`
- DayMap：`scripts/tools/export_daymap_assets.py`

新增同类资产时应补 exporter 和 pipeline test，验证运行时 PNG 是 native 源的整数最近邻导出。

## 角色立绘

重要 NPC 和普通/常客的镜头语言不同，不要混用 prompt：

- 重要 NPC：更强叙事身份、通常用于剧情对话或命运画面。
- 常客/普通客：Tavern 顾客 bust，普通镜头，服务中显示在 `CustomerSprite` 后方。

常客资产当前以 `regular_*` 命名，运行时路径：

```text
assets/textures/characters/regular_<id>_neutral.png
assets/textures/characters/regular_<id>_satisfied.png
assets/textures/characters/regular_<id>_dissatisfied.png
```

对应源和 manifest：

```text
assets/source/tavern/regular_customers/regular_customer_portraits_manifest.json
scripts/tools/export_regular_customer_portraits.py
scripts/test/test_regular_customer_portrait_pipeline.py
```

客群临时客复用常客立绘池，不新增匿名 `guest.png` 占位。

## UI 资产

当前 UI 风格以暗色地下城酒馆、木牌、纸张、烛光、像素字体为主。现有 UI 资产集中在：

- `assets/textures/ui/`
- `assets/textures/daymap/ui/`
- `assets/fonts/fusion-pixel/`

新 UI 优先复用现有组件和纹理风格：

- DayMap 顶部反馈：`GatheringToast`
- Tavern 推理提示：`InferenceReadyNotice`
- Tavern 配方发现：`RecipeDiscoveryNotice`
- Ledger fate notice：`LedgerScreen` 内创建的 overlay

新增 UI 场景前先确认不能复用这些现有反馈通道。

## Contact Sheet 和验证

角色或 UI 批量资产应输出 contact sheet，供人工快速检查：

- 构图是否被裁头/裁手。
- 三表情是否同一身份。
- runtime 尺寸是否符合 Tavern 使用。
- 是否有文字、logo、伪 UI 被烘焙进图片。

运行验证示例：

```powershell
python -m unittest scripts.test.test_regular_customer_portrait_pipeline
```

## 禁止事项

- 不把 AI 原图直接放进运行时 UI。
- 不用程序化装饰代替最终 UI 像素美术。
- 不在图片里生成可读文字。
- 不删除旧 UI，除非替代 UI 已经验证并明确接受。
- 不一次迁移多个 UI 屏幕。
