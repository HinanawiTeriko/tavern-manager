# 游戏总览

> 本文回答“这是什么游戏”。机制细节见 `docs/05_核心机制.md`，物理工作面设计沿革见 `docs/13_合成系统物理重设计需求文档.md`。

---

## 一句话描述

**地下城酒馆** 是一款 2D 吧台模拟经营游戏：玩家白天准备，夜晚在物理吧台上用酒桶、烤架和炖锅服务冒险者，并通过操作方式影响他们的信任和故事。

---

## 概念定位

玩家扮演地下城酒馆老板。客人带着订单、疲惫和未说出口的故事来到吧台前。玩家不只是“选配方”，而是亲手拖动物理材料、摇酒桶、按压烤肉、搅动炖锅，再把成品递给客人。

当前项目的核心方向是：

- **吧台物理手感**：物品有重量、惯性、碰撞和熟度。
- **三容器加工**：酒桶酿饮品，烤架做熟食，炖锅做汤粥。
- **叙事反馈**：配方和上菜风格会改变重要 NPC 的反馈。
- **短循环经营**：白天准备，夜晚服务，打烊结算，逐日推进。

---

## 体验目标

1. **像在吧台后亲手工作**
   玩家从底部快捷栏取材，材料生成物理体，拖、丢、按、摇、搅都会影响产出过程。

2. **容器即动词**
   酒桶、烤架、炖锅要让玩家凭直觉理解“这个材料该怎么做”。想喝的进酒桶，想熟的上烤架，想混成热食的进锅。

3. **操作是态度**
   L3 动作风格把上菜速度分为温柔、平静、粗鲁。重要 NPC 会根据配方记忆和动作风格决定是否讲出更多故事。

4. **小成本叙事压力**
   Ryan 和 Mira 的样板线验证“经营动作影响人物命运”的方向。后续 30 天结构围绕选择、代价和账本结算扩展。

## 核心循环

```text
白天 DayMap
  选择地点 / 采集 / 商店 / NPC 对话 / 携带材料
        |
        v
夜晚 Tavern
  客人点单
  -> 从 ShortcutBar 取材
  -> 酒桶/烤架/炖锅加工
  -> 成品拖入客人投放区上菜
  -> 金币、声望、叙事变量结算
        |
        v
LedgerScreen
  查看账本与当日结果，进入下一天
```
