# 系统总览

## 总体结构

`GameManager` 是当前主流程的唯一协调者。它在 `_ready()` 中创建系统实例，注册当前场景视图，并在 DayMap、Tavern、LedgerScreen 之间传递状态。

```text
project.godot autoload
  GameManager
    EconomySystem
    DayCycleSystem
    DayMapSystem
    GuestSystem
    CraftSystem
    RumorSystem
    AppetiteSystem
    NarrativeManager
    InferenceSystem
    DocumentSystem
    SaveSystem
    RyanSliceSystem
```

UI 场景不直接拥有业务真相。它们通过 `GameManager` 请求动作，再显示返回结果。

## 白天数据流

1. `DayMapView.show_day(day)` 显示地点。
2. 玩家选择地点，调用 `GameManager.visit_day_location(location_id)`。
3. `DayMapSystem.visit()` 处理体力、地点奖励、贴文。
4. `GameManager` 附加剧情、文档、线索、传闻、金币消耗等效果。
5. `DayMapView` 刷新地图，普通采集和传闻使用 `GatheringToast` 做顶部反馈。

传闻由 `RumorSystem.grant_location_rumor()` 记录到当天，夜晚菜单准备读取 `GameManager.get_today_rumors()`。

## 夜晚数据流

1. `TavernView` 注册到 `GameManager.register_view()`。
2. `GameManager` 配置客人数量、今日传闻、昨日回响，打开菜单准备。
3. 菜单确认后 `GuestSystem` 生成重要 NPC、常客或客群客人。
4. `TavernView.show_customer()` 显示客人名、订单、立绘。
5. `BarWorkspace` 生成和拖拽物理物品。
6. 容器产出成品，成品拖入 `CustomerDropArea`。
7. `GameManager.request_serve()` 结算订单、品质、调味、胃口、故事反馈、金币声望。
8. `GuestSystem` 记录当日 guest entry，供结算屏使用。

## 菜单、传闻、客群

传闻不是纯文本提示。它同时影响：

- 菜单准备面板的推荐标签和可能客群。
- `GuestSystem.configure_night()` 的客流权重。
- 常客胃口命中后的记忆和口碑。
- 结算屏的 rumor summary。

匿名客群仍保持 `npc_id == ""`，避免混进常客记忆；但会携带 `guest_group` 和 `portrait_id` metadata，以便 UI 显示复用常客立绘。

## 常客记忆

常客定义在 `data/regular_customers.json`。每个常客有：

- stable id：`regular_*`
- display name
- portrait key
- favorite orders
- trait
- reactions
- neutral/satisfied/dissatisfied portraits

服务命中胃口后，`GuestSystem.record_customer_memory()` 记录记忆，次日菜单准备可显示“昨日回响”。

## 重要 NPC 与推理

重要 NPC 仍由剧情系统驱动：

- `NarrativeManager` 管理变量、好感、结局。
- `RyanSliceSystem` 管 Ryan/Toby/Evelyn 的竖切日程、伪装身份和 fate reveal。
- `InferenceSystem` 管线索、推理题和结论。
- `DocumentSystem` 管证据文档和账本索引。

普通常客和客群不会直接决定重要 NPC 结局，但传闻、推理、文档和 DayMap 事件会给玩家准备信息和经营压力。

## 结算和重写今日

`GameManager._create_ledger_data_for_current_day()` 汇总：

- 金币和声望变化
- guest entries
- 传闻命中摘要
- NPC fate notices
- 下一日警告

`capture_day_start_snapshot()` 在日初记录稳定状态。`restart_current_day()` 恢复该快照，因此会回滚当天购买、采集、传闻、文档、剧情变量、客人记录和事件日志。

## 存档

`GameManager._capture_save_state()` 是保存格式的业务真源，包含 economy、inventory、documents、day_map、rumors、word_of_mouth、inference、narrative、craft、tutorial、ryan_slice、guests、day_start_snapshot、current_day_events。

修改任一系统状态时，应同步检查 save roundtrip 测试。
